在终端tmux会话里按到ctrl+z之后没能彻底杀掉程序

Posted on 2025-05-07 Edited on 2025-05-08 In bugs

ctrl+z是无法终止程序的，它只是把程序挂起来，暂停，但没停掉（所以也不知道这个到底有什么用。。。）

其实一般我是不会用这个的，更可能的是不小心按错了

此时应该终止掉它，也就是切出去看一下是哪个进程，然后kill -9

但不知道为什么，我遇到这个问题的时候，明明kill -9了，之后再nvidia-smi的时候也没看到这个进程了，可是一旦我再用之前那张显卡，程序一开始就会报错out of memory，很明显是之前的程序还在占用着显存

一个解决方法是，在终端运行ps -ax，它大概是会把各种各样的进程都展示出来，并且会显示每个进程具体的运行命令。根据这个命令，可以锁定哪个进程是之前不小心按到ctrl+z的，然后再对应kill -9即可。（这里看的时候，除了我的那个不小心被挂起的程序的进程，还有一个进程，ID是前面那个程序的进程+1，好像是一个类似于指示/标识的作用？应该是因为前一个程序被ctrl+z给暂停了才会有的吧）

反正从操作上，要解决这个问题，就是：
用ps -ax找出对应的进程（以及可能的ID+1的那个附带的进程）
然后kill -9把两个进程都kill掉（如果有第二个进程）
之后再尝试用这张显卡，就能正常用了

登录服务器的时候遇到“无法下载服务器”的情况

Posted on 2025-05-07 Edited on 2025-05-08 In bugs

问题大概就标题所述，具体的症状一般是，（之前能登陆服务器，而现在）用vscode插件登录服务器的时候，输完密码，明明没错，但就是上不去。本来输入密码是在窗口的顶上的，但在那里输入完密码之后，在终端它又要你输入一遍，完了等一会就会报错，说的大概就是这种“Failed to install the vs code server”

这个应该算是相对常见的bug吧，一般的解决方法就是，用其它方式登上服务器（例如，直接用终端，或者用MobaXterm等其它工具），进入到用户的目录下，把.vscode-server这个文件夹给删掉，之后再尝试用vscode插件登录服务器，应该就能登上了（可以参考一下这个博客）

chap1.第三讲 双指针&位运算&离散化&区间合并

Posted on 2025-05-04 Edited on 2025-05-05 In algo

1.双指针

1.1.简介：这个其实更多是一种思想，或者说思路。它的主要作用是利用某种性质，进行优化，通常是把\(O(n^{2})\)的算法优化成\(O(n)\)。这里的性质，可以理解为某种单调性。比如，前面归并排序中，有一步要把两个有序的数组合并为一个有序的数组，其中就用到了双指针：两个指针从两个数组的起点开始，谁指向的元素更小，那个元素就是当前最小的元素，则放到合并后的大数组，并且相应指针往后移动一位。这里用到的单调性就是，两个数组都是有序的，当前指针后面的所有数都是大于等于当前指针指向的元素的，所以每次只需要比较一下当前两个指针指向的元素谁更小，不需要和后面的相比——这里就剩下了时间复杂度。所以最后两个指针都只需要遍历一遍数组，故时间复杂度是\(O(n)\)
