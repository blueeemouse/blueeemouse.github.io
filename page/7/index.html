实现Trie（前缀树）

Posted on 2025-05-08 In algo , 算法面试题

原题链接在这

题目描述：请你实现 Trie 类：

- Trie() 初始化前缀树对象。
- void insert(String word) 向前缀树中插入字符串 word 。
- boolean search(String word) 如果字符串 word 在前缀树中，返回 true（即，在检索之前已经插入）；否则，返回 false 。
- boolean startsWith(String prefix) 如果之前已经插入的字符串 word 的前缀之一为 prefix ，返回 true ；否则，返回 false 。

示例：

输入
["Trie", "insert", "search", "search", "startsWith", "insert", "search"]
[[], ["apple"], ["apple"], ["app"], ["app"], ["app"], ["app"]]

输出
[null, null, true, false, true, null, true]

解释
Trie trie = new Trie();
trie.insert("apple");
trie.search("apple"); // 返回 True
trie.search("app"); // 返回 False
trie.startsWith("app"); // 返回 True
trie.insert("app");
trie.search("app"); // 返回 True

提示：
- 1 <= word.length, prefix.length <= 2000
- word 和 prefix 仅由小写英文字母组成
- insert、search 和 startsWith 调用次数 总计 不超过 3 * 104 次

思路：

这题明确说了是考Trie树，事实上也确实如此，没有别的技巧手法。所以这里就讲一下Trie树是什么，它的原理又是什么

灵茶山艾府的题解写的很好。他的引入也比较好，假设一个字符串只有两种字符，我们可以怎么表示这个字符串？不妨记其中一种字符为a，另一种字符为b，然后，比如字符串是abbaabb，从左到右，我们可以把它看成是一颗二叉树，a就是左儿子，b就是右儿子，这样，这个字符串可以被一个二叉树所唯一地表示。然后，这一题里主要有三个要求（初始化暂且先不管），第一个是insert，也就是向前缀树中插入字符串，那么在这里只有两种字符的例子中，就还是构造相应的二叉树即可；
第二个是search，也就是从根节点出发，搜索一下，有没有待搜索的字符串的对应的二叉树。考虑到有可能一些字符是另一些字符的前缀，例如，abab和ababaa，所以我们考虑给每个节点再附带上一个end属性，用来表征它是不是一个终止节点。从根节点出发，走到一个终止节点，就代表遍历了一个字符串。这样，search的时候，我们从根节点出发，一个一个搜索字符串里的字符，不仅要全部都出现，而且在搜索到了最后一个字符时，对应的节点必须end属性为true
第三个是startsWith，它要判断给定的一个字符串是否为已经出现过的某个字符串的前缀。与第二个要求对比，其实就是，搜索的时候，我们只要求所有的字符一定要出现，且不要求搜到最后一个字符的时候，end属性一定为true，此时我们不关注end属性了，为true或为false都行

那么，现在落实到题目里的要求，会出现所有的小写英文字母，那也就是26种字符，则我们构造的树就变成了26叉树了。至于insert，search以及startsWith三个操作，原理还是没变的

在服务器终端里能正常用conda，但进入tmux会话后无法使用

Posted on 2025-05-07 Edited on 2025-05-08 In bugs

问题描述：

就如标题所述。在外面是能正常用conda的，进入到一个tmux会话之后就不行了。因为在外面可以，所以应该是正常安装上了，那么可能的原因应该是在tmux会话里没激活相应的配置文件之类的
事实上，在tmux会话里用conda相关命令之后，它会提示要先进行conda init，然而如果用conda init，会发现什么结果也没有，它展示了一串配置文件，然后说no change；之后再尝试用conda命令，依然输出说让先进行conda init，卡这了

解决方法：

一个暂时的，但也见效很快的方法，就是在tmux会话里用source ~/.bashrc激活一下bash的配置文件（当然，未必一定是bashrc，要根据具体用哪个shell来决定。不过我倒是几乎全都是用bash的）。至少我尝试之后，这个问题就解决了（至于说暂时，体现在：如果又重新开一个会话，或者在一个会话里新开一个窗口，都会发现还是有之前的问题。这样就需要每次新开一个会话，都用一下source ~/.bashrc，其实倒也还好吧，不能说特别麻烦，但就有点疙瘩的感觉

在终端tmux会话里按到ctrl+z之后没能彻底杀掉程序

Posted on 2025-05-07 Edited on 2025-05-08 In bugs

ctrl+z是无法终止程序的，它只是把程序挂起来，暂停，但没停掉（所以也不知道这个到底有什么用。。。）

其实一般我是不会用这个的，更可能的是不小心按错了

此时应该终止掉它，也就是切出去看一下是哪个进程，然后kill -9

但不知道为什么，我遇到这个问题的时候，明明kill -9了，之后再nvidia-smi的时候也没看到这个进程了，可是一旦我再用之前那张显卡，程序一开始就会报错out of memory，很明显是之前的程序还在占用着显存

一个解决方法是，在终端运行ps -ax，它大概是会把各种各样的进程都展示出来，并且会显示每个进程具体的运行命令。根据这个命令，可以锁定哪个进程是之前不小心按到ctrl+z的，然后再对应kill -9即可。（这里看的时候，除了我的那个不小心被挂起的程序的进程，还有一个进程，ID是前面那个程序的进程+1，好像是一个类似于指示/标识的作用？应该是因为前一个程序被ctrl+z给暂停了才会有的吧）

反正从操作上，要解决这个问题，就是：
用ps -ax找出对应的进程（以及可能的ID+1的那个附带的进程）
然后kill -9把两个进程都kill掉（如果有第二个进程）
之后再尝试用这张显卡，就能正常用了

登录服务器的时候遇到“无法下载服务器”的情况

Posted on 2025-05-07 Edited on 2025-05-08 In bugs

问题大概就标题所述，具体的症状一般是，（之前能登陆服务器，而现在）用vscode插件登录服务器的时候，输完密码，明明没错，但就是上不去。本来输入密码是在窗口的顶上的，但在那里输入完密码之后，在终端它又要你输入一遍，完了等一会就会报错，说的大概就是这种“Failed to install the vs code server”

这个应该算是相对常见的bug吧，一般的解决方法就是，用其它方式登上服务器（例如，直接用终端，或者用MobaXterm等其它工具），进入到用户的目录下，把.vscode-server这个文件夹给删掉，之后再尝试用vscode插件登录服务器，应该就能登上了（可以参考一下这个博客）
