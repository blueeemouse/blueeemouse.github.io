无监督vi-reid的研究意义，主要在于减少在跨模态数据的标注上的开销。因为只要看过红外的图像，就会发现真的很糊，就算是人来，有些图片也很难与可见光图像对应上，除非本人来。这个是比较通用的一个故事吧

一句话总结这篇论文：提出GUR模块，用于提取鲁棒的特征；提出CLU模块，用于平滑伪标签

哪些可以为我所用：也就CLU模块比较有即插即用的潜力。GUR的话，总感觉它过于复杂了……

insight

动机

这篇论文提出，通常的做法注重于缓解跨模态的这种差异，但忽略了camera variation带来的影响，以及二者一起会导致reid更加困难。而它把camera variation以及modality discrepancy二者的相互作用叫做hierarchical discrepancy（所以简单来说，论文认为，以往的方法只解决了hierarchical discrepancy中的一部分，不仅没注意到另一部分，也没注意到这两个部分会相互交织，加大检索图像的难度）
所以它提出一个层次的框架，要解决这个hierarchical discrepancy

方法

1.bottom-up domain learning strategy，核心思想在于，如果两个实例是同一个ID的，那么它们应该有相似之处。然而这里它不是直接用特征相似度来度量，而是间接度量：给定一个memory，它和这个memory里的各个簇的亲和度的分布应该很相似。

2.Cross-modality Label Unification（CLU），做的是一个标签的平滑。（可以认为是为了让标签更加可靠、鲁棒，同时也提高跨模态对应的准确性）。而标签的平滑，可以认为就是对若干的样例的标签进行加权求和。

分析：

1.bottom-up domain learning strategy，包含三个训练阶段，依次是intra-camera，inter-camera和inter-modality training

intra-camera是对多个相机拍到的照片独立处理（具体来说，对于一个摄像头拍到的所有照片，交替进行：聚类分配伪标签，对比损失之经典ClusterNCE。也就是说它捕获的是一些intra-camera similarity）

inter-camera则是会利用一个cross-memory association embedding（CAE）模块，来计算一个图像与多个camera中的簇中心之间的关联性。（注意，此处还是intra-modality的，这里也正好呼应了论文提出的hierarchical discrepancy，它提出的方法也是一层一层来解决问题的）
具体来说，给定一个query，比如它是可见光的，我们会计算它与所有camera的数据中的可见光的簇之间的关联性。每个相机的图片里，都会有可见光的若干个簇（它们都是在前面的intra-camera stage中提取特征并聚类得到的），我们可以先得到这个query和这些簇的相似性，再softmax一下得到一个概率分布。有N个相机的话，我们就有N个概率分布，拼接起来，得到一个大的embedding。我们认为这个embedding就是可以应对camera-variation的

最后的inter-modality stage，则是类似的。此时是在一个跨模态、跨相机的情况下进行的。每个模态下的每个相机的照片中，都会有一些簇（这里的结果是上面inter-camera stage得到的），给定一个相机拍的所有照片，我们还是可以得到query属于这些照片的簇的概率分布。对所有的相机都求一个概率分布，并拼接起来，我们就认为这个embedding是可以应对modality-discrepancy的

2.Cross-modality Label Unification（CLU）

这个模块主要做的就是进行一个加权求和，以求让伪标签更加可靠真实。而这个加权求和的过程，主要是两步，先是跨模态传播标签，再是模态内标签平滑

我们可以认为，之所以需要跨模态传播标签的一步，是为了提高跨模态对应的准确性；而之所以有模态内的平滑的一步，是为了提高标签的准确性和鲁棒性（不难想到，如果我们对一个样例，找出和它比较相似的若干样例，并对这些样例的标签，以相似性为权重，对标签进行加权求和，得到的标签应该是比较可靠的，有种群策群力的感觉）

具体来说，跨模态传播标签，就是，我们聚类得到了红外模态内的簇，因此红外数据有了伪标签；然后我们想通过它传播得到可见光数据的标签。我们的思想是，对每个可见光数据，我们找出离它前k近的红外数据，然后这k个红外数据的标签加权求和一下，就传播得到了这个可见光数据的伪标签。这个过程可以用一个矩阵乘法来表示。
而模态内的标签平滑，也是类似的做法，只不过这时我们找前k个近的数据的时候，是在同一模态内来找的
先是由不同模态的图像来进行（权重就是当前图像与其它所有不同模态的图像之间的相似性分数经过softmax得到的一个概率分布），再是由相同模态的图像来进行（之所以有相同模态的这一步，是因为跨模态的时候，那个相似性分数并不能说十分可靠，而同模态下的相似性分数就比较可信了，毕竟同模态下差异会比较小。这一步就相当于加个保险/平滑吧）

insight：用progressive graph matching的方法来解决跨模态的簇的对应问题，progressive解决了两个模态内簇的数量不一样的问题；转化为graph matching则利用了全局的信息，提高了对应的质量

动机：先前的方法大多是用局部信息来挖掘跨模态对应关系，但忽略了全局信息。其实这段话意思就是说，直接根据特征的距离远近进行跨模态的匹配是不够的，因为这样局限在两两之间的距离远近，没有考虑到更多对象。而论文提出的方法中的Progressive Graph Matching，通过图匹配的方式来利用所谓“全局信息”（因为通过图匹配来构建跨模态的对应关系，就变成一个全局的最优化问题，就不会说单纯的，一看到哪两个跨模态的簇是距离最近的，就匹配上了（这种做法在初期是很可能有问题的））。且两个模态内聚类得到的簇的数量通常也不一样，这也导致跨模态的对齐可能会出问题

方法简单说明

1. 提出Dual-Contrastive Leraning framework(DCL)，操作比较常见，只是对可见光多了一个CA增强

2.提出Progressive Graph Matching(PGM)，把跨模态匹配的问题转化成一个全局的最优化问题（后续工作中借鉴的比较多）

3.提出Alternate Cross Contrastive Learning(ACCL)，为了减小模态间差异，轮流从两个模态出发去拉近实例与跨模态的簇中心之间的距离（后续工作中借鉴思想较多）

方法：

先是一个Dual-Contrastive Leraning framework（DCL，用于获取模态内的关联知识），接着是一个Progressive Graph Matching（PGM method，用于构建可见光和红外数据之间的跨模态联系），最后是一个Alternate Cross Contrastive Learning（ACCL）

1.Dual-Contrastive Learning framework其实就是聚类+对比损失（在无监督vi-reid的设定下，这个损失一般叫做ClusterNCE）。它的核心思想就是，聚类完成之后，对于每个实例，都要让它尽可能接近它所属的那个簇的中心

2.Progressive Graph Matching是通过图匹配的方法来捕获跨模态知识的关联。它解决的问题就是两个模态里的簇的数量不一样的问题，和没有利用全局关系的问题
具体来说，它会先构建两个图，分别对应红外和可见光两种模态。每个图的每个节点都是一个簇中心的特征。而所谓图匹配，就是让每幅图的每个节点都能对应到另一幅图的某个节点上。而关于图匹配结果的评判标准，这里的思想是：匹配到的两个点，它们的特征距离越小，匹配成本就越小；而我们希望最终总的匹配成本尽可能小。由此，可以转化成一个带约束的优化问题，且可以用矩阵乘法表达。这个图匹配问题，有一些现成的算法可以调用，比如Hungarian algorithm

3.Alternate Cross Contrastive Learning（ACCL）。这里涉及到Cross Contrastive Learning（CCL），它就是根据PGM得到的对应关系，会从两个模态出发去拉近实例与跨模态的簇中心之间的距离（举个例子，从红外出发去对齐可见光中的簇中心，就是，给定一个红外实例，我们有它的伪标签，知道它属于哪个红外簇，不妨记为i；根据PGM的匹配结果，我们可以知道红外簇i会对应到哪个可见光簇，不妨记为j，于是我们会用一个对比损失来拉近这个红外实例和可见光簇j之间的距离。而从可见光出发也是类似的，只不过可见光中一半是CA增强得到的图像）
接下来是比较重点，比较精彩的地方。比较容易想到，我们这里的损失就是\(L_{V2R}+L_{R2V}\)，也就是两个模态出发的单向损失之和。但这里有个问题，就是，这种双向损失之和会放大噪声，或者说传播一些错误的信息。因为很显然，初期模型还不够好的时候，提取出来的特征也不够好，进而导致PGM的匹配结果也是不够好的，从而会出现匹配错误，比如一个红外簇和一个可见光簇，其实分别属于ID1和2，但却被匹配到一起了。这时，\(L_{V2R}\)就会拉近红外实例与这个错误的可见光簇之间的距离；而\(L_{R2V}\)则会拉近可见光实例与这个错误的红外簇之间的距离。错误的梯度相当于双倍了，会很影响模型性能。因此，论文提出交替进行，具体来说就是，奇数轮用\(L_{V2R}\)，偶数轮用\(L_{R2V}\)。这样做的好处，一是减少噪声带来的影响（即使初期模型不够好，导致匹配出错，错误信息也只会影响到一个损失，产生一个错误的梯度，更有利于模型在初期的改善，不至于说一开始直接崩了。而只要模型在不断进步，那么模型的匹配结果也会越来越好（注意，每个epoch结束，前面的memory以及匹配结果都会重新计算，不是固定的），相应的ACCL部分也会起到更多正面的效果。换句话说，初期的正确性是很有必要的。所以，一个改进有没有可能可以是，做一个类似warmup的操作，初期的损失权重小一点，经过几轮之后再逐渐增大到1，以确保模型能比较顺利地度过初期。因为很显然，初期是最危险的，有可能直接导致模型不收敛了），二是，我们一轮交替一次，让模型不至于非常明显地偏向于某个模态、过多地从某个模态的角度出发（不妨想一下，两轮交替一次，好像也没问题；三轮交替一次，似乎在后期也还凑合……那么，如果在极限情况，一半的epoch才交替一次呢？可能模型那个时候就会过于偏向一个模态了，导致最终在后面一半的epoch中，进展缓慢，甚至可能不收敛了。但，是否有必要，在模型已经学到一定程度的知识的时候，改变交替的轮数？或者在训练的后期，改成自适应的交替？比如，后面发现模型从可见光到红外做的比较好，但从红外到可见光做的相对差一些，所以就自行更换，用\(L_{R2V}\)？或许也是一个改进的点）

分析

1.Dual-contrastive learning framework，具体来说，会先构建两个memory，对应可见光和红外两个模态；每个模态的memory存的都是聚类得到的所有簇中心的特征。更新方式是一个momentum update，但加进来的特征只是一个这个簇里的某个图像特征
具体到训练方式，在经过聚类得到伪标签后，每个batch中，会对红外采样P类，每类采样K个样本，对这总共P×K个样本都施加ClusterNCE loss（这里是要让每个样本尽可能靠近根据聚类得到的，它所属的那个簇中心）；对可见光，因为这里还会对每个可见光图像进行一个Channel Augmentation（CA），所以原始的可见光图像是随机采样P类，每类随机取\(\frac{K}{2}\)个样本，而这些样本又都会经过CA得到另一半增强后的图像（CA图像），故总数还是P×K，还是对这些样本施加ClusterNCE loss

2.在PGM中，有个细节，就是，论文考虑的是比较一般的情况，即两种模态聚类得到的clusters可能不相同，且假设可见光模态中的cluster数大于等于红外模态中的cluster数。故，一轮下来，其实可能会有可见光中的一些cluster没有红外中的对应cluster。这里就体现出Progressive Graph Matching中的Progressive的由来：它会把剩下的没对应上去的可见光的簇重新构建一个图，而红外部分的图保持不变，再对这两个图进行匹配；不断重复这个过程，直至所有可见光中的cluster都能对应到红外中的某个cluster

这篇论文是基于part-based model来改进的。所谓part-based model，大概就是去提取图像中各种不同部分的特征。可以看看它是怎么提取这些不同部分的特征的，和[[WRIM-Net---Wide-Ranging Information Mining Network for Visible-Infrared Person Re-Identification]]还有[[High-Order Structure Based Middle-Feature Learning for Visible-Infrared Person Re-identification]]之间的差别在哪里？ # 基本概念：1. Mixture-based technique:

- 一种基于混合的技术，用于数据增强。
- 它通过混合图像和相应的标签来合成虚拟训练样本，从而扩大训练集。
- 这种技术可以帮助模型在各种计算机视觉应用中避免过拟合到训练数据。

2. Part-descriptors:

- 局部特征描述符，用于描述人体不同部位的特征。
- 在part-based VI-ReID模型中，会对可见光和红外图像中的不同部位进行特征提取，得到相应的局部特征描述符。
- 这些描述符可以用于后续的识别和匹配过程。

problem：these global feature representations solely focus on the most discriminative part while ignoring the diverse parts which are helpful to distinguish the person identity。（这应该是提取modality-shared feature的方法的通病了，有点类似于去学习一个identity对应的平均模板。但这样太死板了，会遗漏一些细节的、多样的信息）（那么，就像这篇论文着眼的地方——身体部位，这其实就是一种细节信息。如果去捕捉这种细节信息，或许就可以弥补modality-shared feature的短板。类似的，其实[[Channel Augmentation for Visible-Infrared Re-Identification]]这里注重提取的颜色信息也是。那么，如果同时去关注颜色和部位信息，会不会比单一的方法效果要好？进一步的，能不能更自动地，而不是依赖于我们先验设计的着眼点，去提取更多细节信息？提取到了以后，怎么用呢？就把它们和modality-shared feature直接concatenate起来？加起来？用个注意力机制融合一下？蒸馏一下？或许都可以试试，或许有更好的方法）
简单来说，目前用增强方式做VI-ReID的方法， 通常会涉及一些mixture，但没有特别好的方法被设计出来
另外，想利用mixup技术进行增强，辅助vi-reid，但直接应用可能遇到mixup本来就有的问题。（Unlike conventional methods [19,38,44,68,69] that exploit image mixture techniques that yield unnatural patterns, and local patches with only background or single human part,直接应用mixup的时候，就可能遇到这些问题。产生不自然的patterns是很好理解的，看它mixup出来的图像，有些非常的怪异，就可以看出来。重点是说mixup可能产生完全没有人的图像。这就没什么用了，关键它的标签还是人的标签。比如，cutmix，它会从一幅图上切一块下来，放到另一幅图像上。那如果把一幅图的背景给切下来，放到另一幅图像上人所在的区域，就有问题了）（本文的改进则是，不去mixup整个图像，而是去mixup部分，也就是我们提取出来的part）（感觉确实有点道理，就有点类似，同一个人，在同一个场景下，只是穿了不同的衣服，那我们人类应该是很容易就能识别出来的。这里串不同的有点类似不同图像的part进行mixup。那这个方法应该确实有利于去提取一些共性的、具有判别能力的特征）

motivation：希望能设计出一种mixup的方法，避免直接应用mixup的时候会遇到的问题；同时发挥mixup防止模型过拟合的作用，提取出更有效的部位信息
