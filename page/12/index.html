insight

方法：

1.Shallow-Deep Contrastive Learning

对于RGB和IR两种模态的训练数据，要经过两套网络得到各自的shallow和deep feature（所谓shallow feature就是经过一个MSA层，而deep feature就是经过一个transformer）（都是把CLS给提取出来当feature）。它们就组成了四个instance memory。此外，两个模态的shallow/deep feature各自进行聚类（就是在shallow RGB的feature里进行聚类，在shallow IR的feature里进行聚类。。。一共会在四种feature里进行聚类）。至于cluster memory的初始化，就是把对应的那部分特征取均值（比如，要初始化shallow RGB的cluster memory，那就把shallow RGB的所有特征取均值。其余类似）

cluster memory的优化主要是动量更新

因为在四种特征里都进行了聚类，比如在shallow RGB feature里进行聚类，得到5类。现在这些类就是伪标签，我们要用它来进行对比学习。这里的对比学习主要是优化前面提取shallow/deep feature的网络的，也就是为了更好地提取特征

2.Collaborative Neighbor Learning

大体上做的事就是把各种特征进行对比，也有跨模态的特征对比，所以它一直强调cross-modality supervision

特征的相似度用余弦相似度来度量。然后shallow/deep有两种选择，RGB/IR两种选择，又因为是两个特征之间进行度量，理论上会有16种不同种类的特征进行相似度度量。然而要注意的是，shallow/deep的选择一定是同步的，即，要么同时度量两个shallow feature，要么同时度量两个deep feature。因为度量特征的最终目的是希望拉近某些特征的距离，但我们并没有必要去拉近某个deep feature和shallow feature之间的距离（我们的本意就是想分别利用shallow和deep feature各自含有的不同信息，这样拉近距离反而可能造成一些问题）。所以实际是有8种不同特征的搭配来进行度量。
首先注意，一个输入图像（不论是RGB还是IR），都会有shallow和deep两种feature。
其次，计算损失的时候，是先给定一个batch的query。每个query是一个图像，它会有上面说的两种特征。然后，对每个query，我们可以计算它的同一个模态的邻居，也可以计算它的不同模态的邻居。不妨以query为RGB图像，要计算它的不同模态（即IR模态）的邻居为例。我们先拿到该query的shallow feature，然后让它和所有IR模态的shallow feature进行相似度度量（因为我们这里要计算的是不同模态的邻居），之后根据相似度高低，进行一些筛选（选出多少则是由一个超参数γ决定），得到一些IR模态shallow feature。再取出该query的deep feature，让它和所有IR模态的deep feature进行相似度度量，取一些相近的deep feature。两组featured的交集，即该query的不同模态的邻居。其它几种邻居计算方法类似。
得到每个query的邻居后，开始计算对比损失。大体上分成四个部分，vv、vr、rv、rr（v代表RGB，r代表IR）。不妨以vv的损失计算过程为例。首先，vv损失也会分成shallow和deep两部分。然后shallow部分就是batch里的每个query都计算它和它的shallow邻居之间的对比损失，再取均值。deep部分类似。二者相加（shallow部分有个超参数）得到vv损失。其余几个损失计算方法类似。
整个方法的损失就是两个部分，一个是这里的四部分对比损失之和，另一个就是shallow-deep contrastive learning里的损失

如果说3.1的Shallow-deep Contrastive Learning主要是特征之间自己对比，并且都是同一模态下的，那么3.2的Collaborative Neighbor Learning就是query和neighbor之间的对比，并且有跨模态的对比

3.Collaborative Ranking Association（这一步是在利用上面的特征进行推理）

这一步的执行顺序应该是在聚类完之后，根据下面两个假设，对聚类的伪标签进行了一些平滑操作。之后才是1.里的Shallow-deep contrastive learning（那里的contrastive learning依赖于伪标签）

它是基于两个假设提出的：
一是，在两个模态之间用shallow/deep feature进行相似度度量，之后进行排名，这两个排名应该尽可能一致
二是，同一个模态内shallow/deep feature进行相似性度量，并进行排名，这个一致性会相比上面的更好

这里会有两个步骤

一个是Inter-modality Ranking Association
给定一个RGB图像，我们有它的shallow和deep feature。用shallow feature和所有的IR图像的shallow feature进行相似性度量，得到一个排序；deep feature和所有的IR图像的deep feature进行相似性度量，也得到一个排序。我们假设deep feature得到的排序里有N个结果（也就是假设IR图像一共有N个），统计相似性从高到低的每个结果，它的标签和多少个shallow feature的排序里的结果的标签相同。这里从简，我们记deep feature的第k个结果对应和shallow feature里\(n_{k}\)个结果有相同标签。最后，这个RGB图像的标签就应该是最大的\(n_{k}\)对应的那个标签。最终对每幅图像都做这个操作，得到一个标签矩阵，每一行是一个独热向量

第二个是Intra-modality Collaborative Smoothing（这个还是很合理的。毕竟光用deep feature得到一个相似度矩阵，其实可能不太够，因为前面说了，shallow feature还是可能提供一些信息的，也要利用起来。但这里综合利用二者的方式是分别求得相似性矩阵，然后相加。有没有其它的综合方式？直接融合可以吗？）
我们用所有RGB图像的shallow/deep feature，两两计算相似度，得到两个矩阵，两个矩阵相加，之后每一行将前五大的数置为1，其余置为0（这里想做的就是根据shallow feature similarity和deep feature similarity，综合考量，得到每一个样本可能和哪几个样本相似性较高）。得到这个矩阵后，用这个矩阵左乘上面的到的标签矩阵，又得到一个矩阵，它的形状是N * C（N是RGB图像的样本数量，C是IR图像的类别），每一行最大的元素对应的列数就是这个样本的最终的标签

1.intro

Person Re-IDentification (ReID)，是一个重要课题。主要应用在智能监控系统上。其中有个问题，现有的模型大多是只基于图像，即只有一个模态。这背后的问题是：在光线不好的时候，图像可能质量很差，导致模型表现很差。这极大阻碍了24 * 7的监控系统的构建。而注意到红外成像即使是在光线不好的情况下也能生效，加上很多摄像头是有图像/红外线两种模式的，而且可以智能地根据环境光线进行切换，因此visible-infrared cross-modality person re-identification（VI-ReID），即基于image和红外成像的行人重识别是很有必要的

在此之前的行人重识别的综述中，关于VI-ReID的描述都不够详细，而少有的专门介绍VI-ReID的综述，它的分类比较粗糙，而且甚至有的方法都不能归到任意一个类里。因此，有了这篇综述，它希望能给出合理的分类，清楚的讲述各个方法的动机和优缺点，提供一个big picture

2.definitions and challenges

关于VI-ReID的定义：它也是要进行一个匹配，即将一个输入的probe/query image，与gallery images里的N张图像进行匹配，但二者的模态不同。可能probe是RGB的，gallery images是IR的，也可能反过来

关于VI-ReID的困难：

因为涉及两个模态，自然的，两个模态之间的差异是需要考虑的。这也就是所谓的cross-modality variations。比如，RGB图像显然有丰富的颜色、纹理等细节信息，但IR图像一般就没有。具体来说，可能导致这个问题：several misalignment problems across different modalities for the same identity。也就是说，同一个人，在不同模态里我们却可能匹配不出来。举个例子，一个行人，我们显然希望它的RGB图像能和它的IR图像匹配上，毕竟是同一个人。但因为两个模态之间的巨大差异，可能不太能匹配上。甚至可能有这种情况：probe是A的一个RGB图像，然后gallery images是一些行人的IR图像。然而进行匹配的时候，却把probe匹配到了gallery images里B的IR图像了。这也是因为，我们要进行两种模态之间的匹配，那么用到的信息应当是两个模态共有的。而RGB模态和IR模态共有的信息显然是比单一的RGB/IR模态的信息要少的。那么，信息量更少，任务却不变，显然难度更大了

另外，每个模态内自己固有的差异也还是存在的。就比如说，即使都是RGB图像，因为各个图像的视角不同，可能有些遮挡之类的，图像里人的姿态可能也不一样，那么光在RGB modal内进行重识别依然有难度。这其实也是传统的行人重识别中就有的问题。

3.datasets and evaluation metrics

datasets：有两个常用的，一个是SYSU-MM01，一个是RegDB（详细信息看论文吧，已经说的挺清楚的了）

metrics：两个常用的，一个是Cumulative Matching Characteristics（CMC），另一个是mean Average Precision（mAP）（可以参考博客）

4.methods

4.1. modality-shared feature learning

这个方法主要聚焦于提取出不同模态共有的特征。比如，可能RGB模态和IR模态都有比较好的轮廓的信息，于是可以尝试提取两个模态里都有的轮廓，依靠轮廓来进行行人重识别。其中，这种方法又有三种不同实现方法

4.1.1. feature projection

这类方法有三种常见架构：one-stream structure，two-stream structure和asymmetric fully-connected layer structure
one-stream structure就是用一套卷积网络去提取两个模态的共同特征（用一套卷积网络，显然参数是共享的）。此时两种模态的输入是可以混合的
two-stream structure则是先用两套卷积网络和全连接层去分别提取两个模态自己的特征（显然这里参数是不共享的），然后再用一个公共的卷积网络，把提取出来的特征变换到一个公共的特征空间里（也就是说，我们期望这个特征空间是两个模态共有的）
asymmetric fully-connected layer structure则是几乎全程都是共享参数，前面和one-stream很像，但两个模态分别输入，且最后会用两个不同的全连接层得到两个不同输出

问题：

从一个view出发计算得到的code（或者说cluster assignment）到底是什么？其形状为？怎么计算得到？它的含义是什么？

code就是指样本属于各个prototype的概率分布。
先说一下它的求解顺序。实际上，求解它之前需要先计算特征和各个prototype之间的相似度。相似度就是余弦相似度（特征和prototype都会被归一化）。举个例子，一个batch data的特征，形状为（batch_size, dim），prototype的那个矩阵，形状为（dim，num_prototypes），二者进行矩阵乘法即可。求解出这个还不算完，这个还不是最终的code。并不是因为还没归一化，而是因为我们对它有一定的约束，即我们希望batch data均匀地分布在各个prototype之间，否则有可能模型求得平凡解，就是把全部样本都输出为一个表征，然后每个表征从属于各个prototype的概率都均等。而加了约束，就相当于求解一个优化问题，所以论文里提到要用Sinkhorn-Knopp算法进行迭代求解，并且这部分操作是不计算梯度的，也就是说，每一个训练的step里，我们会计算cluster assignment，但它只是用于计算梯度，我们不会去反向传播优化cluster assignment。
在代码里，code是一个列表，每个元素是batch data的一个augmentation的code，也就是说，每个元素是一个张量，其形状为（batch_size, num_prototypes）。它的计算并不是通过对比损失得到，而是要用一个Sinkhorn-Knopp算法迭代求解一下，

swav这么对比，其出发点是？合理吗？相比于之前的直接对比特征，有什么区别？优劣何在？

出发点就是，如果是同一图像的不同视图，它们的表征应该尽可能相同，这样的话由一个视图a1，经过一个操作p1得到一个结果r1，然后我用另一个视图a2，也去做这样的操作p，得到的结果r2应该和上面那个结果r1很相似。具体到swav，它觉得，由一个视图a1的特征，计算得到prototype（但实现的时候，就像上面说的，只是求得一个概率分布），应该和用a2的特征计算得到的prototype很相似

swav计算对比损失和优化prototype（指优化那些prototype模板，不是计算code）的过程的顺序是怎样的？谁先谁后？还是同时？

如果只是优化那个模板矩阵（每一列表示一个prototype）（在代码里也就是一个不带bias的Linear层），那么其实就是计算完对比损失和分类损失之后，反向传播的时候就在优化了。比较令人困惑的点应该是，code其实并不需要优化。这点在第一个问题的回答里已经说明了

求解code被转化成了一个优化问题？这个优化问题的求解用到iterative Sinkhorn-Knopp算法？这个算法的流程和细节是怎样的？

是的，求解code就是在我们的约束下的一个优化问题。至于用到的这个算法，只能说没啥兴趣去了解。而且，个人感觉，这个所谓的用prototype去enforce样本之间的一致性，好像不够简单本质，只能算个trick。它之所以提点了，估摸着也是multi-crop的功劳居多吧

在进行数据预处理的时候，一般都会有一个类似train_transform这样的数据增强的“列表”，它里面包含了一系列操作。通常是调用库里的那些操作。但如果需要自己定义一些预处理操作，则需要自己写一个类。一般有这些要求：

1. 类定义：自定义的数据操作一般是定义为类（如 CIFAR10Policy），并且实现了 __call__ 方法。__call__ 方法使得类的实例可以像函数一样直接被调用。

2. 实现 __call__ 方法：当类的实例通过 transforms.Compose 被应用到图像时，实际上是调用了该类的 __call__ 方法。因此，在 __call__ 方法中，你可以定义自定义的图像变换逻辑。

3. 不变性要求：数据增强和变换操作应该是“无副作用”的，这意味着不应直接修改输入图像，而是返回一个经过变换的新图像。

4. 可与其他操作链式使用：自定义的数据操作通常应该与 PyTorch 的其他数据转换操作（如 transforms.ToTensor() 或 transforms.Normalize()）兼容，因此要确保其输入输出类型与其他操作一致。

- CutMix 是一种数据增强方法，它将两张图片进行随机部分的组合，同时将两张图片的标签按比例加权。例如：

- 一部分图片 A 和一部分图片 B 被切割组合，输出图像的标签是 A 标签和 B 标签按图像的组合比例加权。

- rcpaste（Random Cut Paste）是一种随机增强方法，通常涉及在图像中随机剪切部分图像并粘贴到其他区域，增加图像的多样性。

- MixUp 也是一种数据增强技术，涉及将两张图片的像素和标签按某个随机比例进行线性混合。它通过图像的加权平均生成新的训练样本，其对应标签也按相同比例混合。例如，图像 A 和 B 混合后新图像的标签为 λ * 标签 A + (1-λ) * 标签 B，其中 λ 是随机生成的混合比例。

- 这几个操作实在是有点太像了。主要区别可能在于，random cut paste是用自己的图像完成操作，而cutmix和mixup都是用到两幅图像。cutmix是部分的完全组合，mixup是整体的部分组合

先讲讲把transformer照搬到cv的困难。其实就在于注意力机制计算复杂度是平方级别的，所以输入序列不能太长。而如果我们要用transformer处理img，也需要把img转化成类似nlp那边的序列类型的输入。然而如果是朴素地把img展开，基本都超了通常能接受的seq长度。例如，bert能处理的seq长度也就512，而你大一点的img的单边的像素值都超了512了；假如是一个224*224的img，其实不算太大了，但展开的话长度就是50172了，完全顶不住

这篇工作就是vit。它的模型部分就是把nlp那边的transformer几乎照搬过来，做尽可能少的改动（在此之前，研究注意力机制与cv的工作，主要是聚焦在两方面 ：一是在现有的cnn架构中把一部分卷积操作换成自注意力机制；二是研究把transformer搬到cv之后，怎么降低其计算量，研究魔改一下注意力机制。但它们这些方法因为没有硬件加速，无法扩展得很大，最终效果应该还是不够好）。而vit想尽可能照搬nlp那边的处理方法，看能transformer能在cv领域scale到多大，效果能有多好。有种大力出奇迹的感觉。那它为了解决序列长度过长，而又不想改变注意力机制，就把img划分为多个16*16的patch，每个patch当成是nlp那边一个token的embedding。划分完，seq长度就正常多了，有可能训练了

感觉这篇论文其实所谓创新性真不高，就是敢于训大模型，而且也有充足的资源。此外就是实验详实，考虑了很多细节，并都做了对比实验。
但从结果上看，确实是连接了cv和nlp两个领域，也给多模态带来了很大的影响

method部分没什么好说的，它写的已经比较清楚了。

比较值得玩味的就是所谓的inductive bias。cnn里主要是locality和translation equivariance（现在终于有点理解是什么意思了。假如我们认为邻近像素比较有可能相近，这就是locality的bias，姑且当成是一种先验知识。cnn的架构也满足这个locality，所以它处理img的时候也会认为邻近像素会比较接近）

另外，文章里多次提到inductive bias

一是说为什么transformer应用到cv领域的时候，数据集和模型小了效果不够好：因为transformer比cnn少了一些inductive bias，也就是少了一些先验知识
