在排序数组中查找元素的第一个和最后一个位置

Posted on 2025-07-15 In algo , 二分算法 , 二分查找 , 基础

原题链接在这

题目描述：

给你一个按照非递减顺序排列的整数数组 nums，和一个目标值 target。请你找出给定目标值在数组中的开始位置和结束位置。

如果数组中不存在目标值 target，返回 [-1, -1]。

你必须设计并实现时间复杂度为 O(log n) 的算法解决此问题。

示例 1：

输入：nums = [5,7,7,8,8,10], target = 8

输出：[3,4]

示例 2：

输入：nums = [5,7,7,8,8,10], target = 6

输出：[-1,-1]

示例 3：

输入：nums = [], target = 0

输出：[-1,-1]

提示：

  • 0 <= nums.length <= 10^5

  • -10^9 <= nums[i] <= 10^9

  • nums 是一个非递减数组

  • -10^9 <= target <= 10^9

    思路：

    其实这道题是二分查找里最经典的题之一了，之前也写过，但过一段时间再回来看， 就又写不出来了，只能说确实还没有深刻理解

    不过这次写，有一些其它的琐碎的想法。首先是，这题竟然允许nums为空，这个实在是没注意，导致一开始测试的时候有类似“引用空指针”的报错
    另外，我们之所以在这一题上要用二分查找，不仅是因为题目说了数组是递增的，方便我们使用，更根本的原因在于对时间复杂度有限制，要求一定是\(O(log\ n)\) 的，这就限制我们不能用朴素的、\(O(n)\)的方法。如果能用\(O(n)\)的方法，也就不需要绞尽脑汁写二分了。只能说不要想当然地上来就用xxx算法，而是要真的理解，并根据题意选择合适的工具来使用

    回到题目本身的话，我们可以用两次二分查找，一次查找左边界，一次查找右边界
    因为沿用了之前chap1.第一讲 快排&归并排序&二分查找里面的思路，所以主要的问题，或者说难点，还是在于l、r的赋值。具体可以看那一篇
