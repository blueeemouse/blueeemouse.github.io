venv简介

Posted on 2025-07-03 Edited on 2026-01-14 In 配置相关

简单介绍一下这个venv。实习期间的项目用到了。之前python的虚拟环境都是用的conda来管理的。这次因为懒得装conda，就用了venv。使用下来，感觉venv其实很适合仅用于一个项目环境的配置，主要它直接是一个文件夹，里面放了所有的库，给人很直观很简单的感觉。真别说，很轻量，感觉不错

常用命令：

uv venv，使用之后会在当前目录下创建一个.venv/，它就是我们这个项目的虚拟环境的目录（注意，虽然这个虚拟环境可以改名字，但大家一般都默认用这个.venv了。甚至于git都知道这个东西不需要提交，即使没在ignore文件里写，也不会提交）（也因此，一个项目，如果我们创建了一个venv环境，然后多套内容，希望用到多个git分支管理，那我们的venv是可以在这个项目下的不同git分支里复用的，不会改变）

如果要指定创建的环境的python版本，一般是uv venv --python 3.11这个命令。后面的python版本得确定uv里有才行，或者说得确认uv能看到。那么需要执行uv python list，确认里面有这个python版本。如果没有，就得执行uv python install 3.11这个命令来安装指定python版本

若是直接用uv venv创建，那么它会根据据当前终端里的python版本直接创建

其它问题：

如果我们在ide里（例如vscode），想在里面指定环境（不是在终端指定，那个很简单；是在ide里指定。这样指定之后，我们才能基于那个环境看我们的代码。那比如，我们右键想看看某个库的源码，如果ide识别到的是默认的环境，没装这个库；虽然我们的项目环境装了，但还是无法跳转），思路就是在ide里手动选择python解释器。具体到vscode的话，可以用Ctrl + Shift + P，然后在唤出的输入栏里输入Python: Select Interpreter（不一定是完全一样的字，反正是选择interpreter就对了），然后在里面选就行

但如果是venv，它可能识别不到。这时就需要我们手动输入路径。也就是输入venv的路径

完了之后，正常应该就显示我们想要的环境了。如果不行，试试reload window
