最长公共子序列

Posted on 2025-07-20 Edited on 2025-07-21

原题链接在这

题目描述：

给定两个字符串 text1 和 text2，返回这两个字符串的最长 公共子序列 的长度。如果不存在 公共子序列 ，返回 0 。

一个字符串的 子序列 是指这样一个新的字符串：它是由原字符串在不改变字符的相对顺序的情况下删除某些字符（也可以不删除任何字符）后组成的新字符串。

  • 例如，"ace" 是 "abcde" 的子序列，但 "aec" 不是 "abcde" 的子序列。

    两个字符串的 公共子序列 是这两个字符串所共同拥有的子序列。

    示例 1：

    输入：text1 = "abcde", text2 = "ace"

    输出：3

    解释：最长公共子序列是 "ace" ，它的长度为 3 。

    示例 2：

    输入：text1 = "abc", text2 = "abc"

    输出：3

    解释：最长公共子序列是 "abc" ，它的长度为 3 。

    示例 3：

    输入：text1 = "abc", text2 = "def"

    输出：0

    解释：两个字符串没有公共子序列，返回 0 。

    提示：

  • 1 <= text1.length, text2.length <= 1000

  • text1 和 text2 仅由小写英文字符组成。

    思路：

    首先得对一些基本的术语进行说明。一般，子数组、子串，这种说法，它们是连续的。而子序列则未必（比如这题的子序列，按照它的定义，就是可以不连续的）

    下面来看这道题。因为这题的子序列的定义，看重的是字符之间的相对顺序，所以两个字符串s1和s2的最长公共子序列lcs，其构造可以看成是，两个字符串各自都选出了一些字符，筛掉了一些字符。不妨以题目里的示例1为例，它的最长公共子序列是"ace"，可以看成是text1里选了第1、3、5个字符，以及text2里选择了所有字符。这样，我们就可以转而去讨论每一个字符是否被选择了。故，用动态规划的话，假设我们想知道text1的前i个字符组成的子串于text2的前j个字符组成的子串的lcs（longest Common Subsequence，最长公共子序列的缩写，下均用lcs指代最长公共子序列）的长度（不妨记为k），那么，可以有这样的递推关系：

    （这里为了记号的方便，我们考虑引入一个二维数组lcs，其元素lcs[i][j]表示以text1的前i个字符组成的子串与text2的前j个字符组成的子串的lcs的长度）

    如果text1的第i个字符和text2的第j个字符是相同的，则直观上想，我们为了获得最长的公共子序列，似乎应该把text1的第i个字符和text2的第j个字符都选上，此时就可以把问题转换成求text1的前i-1个字符组成的子串和text2的前j-1个字符组成的子串的lcs长度（因为得到了这个最长公共子序列之后，它们都拼上最后的text1的第i个字符，以及拼上text2的第j个字符就得到了最终的最长公共子序列），最后加个1就行，即有递推式：\[lcs[i][j]=lcs[i-1][j-1]+1\]

    如果text1的第i个字符和text2的第j个字符是不同的，那直观上想，我们要获得最长的公共子序列，就不能同时选上text1的第i个字符和text2的第j个字符了，只能是选一个。则有递推式：\[lcs[i][j]=\max(lcs[i][j-1],lcs[i-1])\]

    上面的递推式虽然很简单，但其实证明上是有些不严谨的。比如，当text1[i]==text2[j]时，为什么在取lcs的时候，一定会把text1[i]和text2[j]都选上呢？就不能有一个不选吗？（注意，二者都不选显然是不行的，因为这样得到的lcs一定会比都选上时的lcs要小）有没有可能，比如我不选text1[i]，然后text2[j]就能和text1里的其它字符对应上，最终反而lcs会更长呢？实际是可以严格证明的，不过还挺麻烦

    给一些记号，以便证明。考虑记两个字符串分别为s和t。我们现在考虑，当s[i]==t[j]，lcs[i][j]=lcs[i-1][j-1]+1>lcs[i-1][j]（左式表示s[i]和t[j]都选上，右式表示不选s[i]，只选t[j]。至于另一种的只选s[i]，不选t[j]，基本是对称的，证明方法类似。

    我们的思路其实是，先证明，当s[i]只在s里出现一次的时候，上面的结论显然成立；当s[i]在s里不止出现一次的时候，则需要额外证明

    首先，因为lcs[i][j]的含义是，s的前i个字符组成的子串与t的前j个字符组成的子串的lcs，所以我们考虑lcs[i][j]的时候，其实不需要考虑s在i个字符后面以及t在j个字符后面的内容了。所以简单起见，我们可以直接假设s一共就i个字符，t一共就j个字符

    接下来，我们先证明，当s[i]只在s里出现一次的时候，结论成立

    此时，如果我们不选s[i]，因为s[i]只在s里出现过一次，但我们在t里面又选了t[j]，而t[j]等于s[i]，则会出现，t[j]在s里没有人对应了（只有一个s[i]可以对应，但是这里我们又不选）。所以，其实相当于，这种情况下要是不选s[i]，那也就不能选t[j]了，这不就退化成了lcs[i-1][j-1]吗，而且s[i]和t[j]相等，所以这肯定就是白白小了1，所以结论成立，此时应该就是s[i]和t[j]都选才是最优的

    然后，我们来证明，当s[i]在s里出现不止一次的时候，结论还是成立

    此时，我们不选s[i]了，但是又选了t[j]，所以t[j]就必须对应到s[i]在更前面一点出现的位置了（因为此时已经明确，s[i]在s里出现不止一次）。为了便于理解，我们不妨假设，s[i]就是字符a，因此t[j]也是a。不妨再假设，t[j]对应到的a出现在s里的位置为p，则此时相当于变成了lcs[p][j]（t[j]已经是t的最后一个字符了，它对应到s的哪个字符，s取子序列的时候也就终止到那个位置），且根据假设，应该有lcs[p][j]>lcs[i][j]
    不妨再记lcs[i][j]为x，则应有lcs[p][j]>=x+1；而因为s[p]==t[j]，当我们同时把s[p]和t[j]去掉，考虑lcs[p-1][j-1]的时候，又有lcs[p-1][j-1]>=x
