多数元素

Posted on 2025-05-13 In algo , 算法面试题

原题链接在这

题目描述：

给定一个大小为 n 的数组 nums ，返回其中的多数元素。多数元素是指在数组中出现次数 大于 ⌊ n/2 ⌋ 的元素。

你可以假设数组是非空的，并且给定的数组总是存在多数元素。

示例 1：

输入：nums = [3,2,3]

输出：3

示例 2：

输入：nums = [2,2,1,1,1,2,2]

输出：2

提示：

- n == nums.length

- 1 <= n <= 5 * 10^4

- -10^9 <= nums[i] <= 10^9

思路：

题目还是比较友好的，确保了数组非空以及总是有多数元素

最朴素的思路是（主要是受到[[字符串中的第一个唯一字符]]这一题的影响吧，觉得有点像，所以往这方面想了一下），遍历一遍数组，然后把相同的元素值映射到某个集合的同一个键，而值就是这个元素值出现的次数。这样，我们最终找到出现次数最多的那个即可。但由于对集合以及哈希不是特别熟，其实这个方法不太能实操起来。有一个特别巧妙的思路，而且特别适合这道题，叫摩尔投票。详细的讲解不妨看一下这里，讲得比较明白了。它主要是假设，题目里提到的，出现最多的数，如果它能投+1的票，其余元素投的是-1的票，则最终所有票加起来一定是大于0的（因为已经保证了，出现最多的那个元素，出现次数大于⌊ n/2 ⌋嘛（这个大于很关键）

但总结起来一句话就是，我们从前向后遍历元素时，不同的元素就抵消，这样抵消下来一趟，还剩下来的一定是众数，也就是我们要找的数。但怎么实现这个元素的抵消呢？我们可以用计数来表征。从前往后看，第一个元素，一开始只出现了第一次，cnt变量就赋为1，再看第二个元素：如果第二个元素和第一个元素相同，那cnt++，此时cnt是2，第一个和第二个的元素值就是当前的众数。如果它们真的是全局的众数，则后续按照这个规则继续加减，cnt一定大于0（因为此处的众数一定多于一半），而如果它们其实不是全局的众数，真正的全局众数在后面，那按这套规则往后走，一定会先出现cnt被众数减到0的情况，之后我们继续往后抵消，cnt最终还是可以保证大于0）（唉，这段没说清楚，感觉理解不深刻……）；如果第二个元素和第一个元素不同，则cnt--。此时cnt变成0了，则相当于第一个和第二个元素已经抵消了，之后我们只需要看后面的元素进行抵消即可。所以当前众数就可以定为第三个元素
