打乱数组

Posted on 2025-05-18 In algo , 算法面试题

原题链接在这

题目描述：

给你一个整数数组 nums ，设计算法来打乱一个没有重复元素的数组。打乱后，数组的所有排列应该是 等可能 的。

实现 Solution class:

- Solution(int[] nums) 使用整数数组 nums 初始化对象

- int[] reset() 重设数组到它的初始状态并返回

- int[] shuffle() 返回数组随机打乱后的结果

示例 1：

输入

["Solution", "shuffle", "reset", "shuffle"]

[[[1, 2, 3]], [], [], []]

输出

[null, [3, 1, 2], [1, 2, 3], [1, 3, 2]]

解释

Solution solution = new Solution([1, 2, 3]);
solution.shuffle(); // 打乱数组 [1,2,3] 并返回结果。任何 [1,2,3]的排列返回的概率应该相同。例如，返回 [3, 1, 2]
solution.reset(); // 重设数组到它的初始状态 [1, 2, 3] 。返回 [1, 2, 3]
solution.shuffle(); // 随机返回数组 [1, 2, 3] 打乱后的结果。例如，返回 [1, 3, 2]

提示：

- 1 <= nums.length <= 50

- -10^6 <= nums[i] <= 10^6

- nums 中的所有元素都是 唯一的

- 最多可以调用 10^4 次 reset 和 shuffle

思路：

因为数组里没有重复元素，实现起来会相对简单一点（但如果是暴力方法的话，即使是有重复元素，好像也是一样做吧？）

看一下要求，初始化，可以想到大概是把初始传入的数组给保存到某个成员变量里，就类似python里的self.xxx = input这种格式，只不过cpp太久没用了，这一点也是真的忘了具体语法了……
reset的话，最简单粗暴的方法当然也是直接把初始数组存一份，调用reset的时候直接返回存下来的这份结果就好
shuffle的话，要随机打乱，重点就是题目里提到的，要求所有排列应该是等可能的。这里数组里是没有重复元素的，那么，如果我们这样去构造：对于结果数组，第一个元素从当前数组里随机挑一个，第二个元素从当前数组里剩下的元素中随机挑一个，以此类推，也就是我们遍历位置，元素随机挑，那算一下，不管是哪种排列，概率都是一样的（不妨举个例子，一共5个元素，1-5，要随机打乱它。按照上面这个方法，对于任意一种排列结果，概率一定为\(\frac{1}{5} \times \frac{1}{4} \times \frac{1}{3} \times \frac{1}{2} \times \frac{1}{1}=\frac{1}{120}\)）

代码：

思路倒是讲的比较清楚了，可能更多是复习一些语法细节

第一点就是，类里定义了成员变量后，不管是public还是private，都可以在成员函数里直接调用（无论是public还是private的函数都是可以直接用的）。在这里定义到private，主要是出于一种习惯，因为定义到private的变量，就不会被外界直接调用和改变，相当于保护了数据的安全；而如果把成员变量定义到public，外界是可以通过类实例.变量这种形式来调用和改变的
还有个细节就是，虽然成员变量可以直接在成员函数里用，但如果出现了重名的现象， 那需要用this->成员变量这种形式来调用。比如，如果我们在private里定义成员变量，当前数组的名字定义成nums，则在初始化的时候，因为传入的参数数组也是叫nums，如果直接nums=nums，就是参数自己给自己赋值了，那什么也没改变，我们没给成员变量赋值。此时就需要这样赋值：this->nums=nums

第二点就是，在shuffle函数里，我们每次挑选元素的时候，都是从剩下的没挑过的元素里随机选。因为要标记哪些挑过、哪些没挑过，所以这里用到了unordered_set来存储索引，还在这个集合里的就是还没被挑过的。而随机挑选一个索引之后，我们需要确认一下这个索引是不是没被挑过的，是的话就可以用，不是的话就要重新生成一个索引
用到的rand函数，它是在cstdlib这个c++标准库或者在stdlib.h头文件里，返回的结果是一个在0到RAND_MAX之间的伪随机整数（之所以是伪随机，因为用的生成的算法是伪随机算法，主要是线性同余法）。这里我们生成一个随机数之后，再对数组长度取个模，就能得到索引了
此外，代码里还用到了unordered_set的insert函数和find函数，insert函数就是插入一个元素，当然它其实是会根据插入结果返回一个pair的，简单复习一下，pair的第一个值是指向新插入的元素的迭代器，或者如果原来集合里已经有这个元素了，则这个值就是指向已有元素的那个迭代器；而第二个值则是表示插入成功还是失败，是个bool变量，true的话就说明集合里原来没有这个元素，现在插入成功了；false的话则表明原来集合里已经有这个元素了，插入失败了（或者说等于没插入）。至于find函数，这里我们是用来判断一个元素是否已经在unordered_set里出现过，find函数返回的是迭代器，如果指向的是end()，则表明这个元素原来不存在，否则返回的就是集合里指向这个元素的迭代器
