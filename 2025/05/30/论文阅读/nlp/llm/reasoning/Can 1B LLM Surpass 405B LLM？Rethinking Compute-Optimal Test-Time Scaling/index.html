Can 1B LLM Surpass 405B LLM？Rethinking Compute-Optimal Test-Time Scaling

Posted on 2025-05-30 In 论文阅读 , llm , reasoning

1. abstract

这篇论文关注的是，在不同的policy models、PRMs（Process Reward Models）以及problem difficulty下，是否有一个最优的tts策略？
以及，tts方法的扩展上限在哪里？小模型能否通过tts，在推理方面胜过大模型？

结果表明，tts的策略选择极大地依赖于policy model，PRM以及problem difficulty；

另外，小模型确实能通过tts的扩展，在推理方面击败大模型

综上，tts是一个在提高llm推理能力上比较有前景的方法

问题：什么是tts？它是怎么运作的？
什么是policy models？看论文里的脚注，好像是指推理情景下，生成答案的llms？
那什么是Process Reward Models？

2. Setup & Preliminaries

2.1. Problem Formulation

首先，我们把推理问题放到马尔可夫决策过程（Markov Decision Process，MDP）的框架下。它的五元组是$(S,A,P,R,\gamma)$，分别表示状态空间、动作空间、转移函数、奖励函数以及折扣因子

给定prompt（对应到推理问题，也就是给定问题），大模型（或者说策略模型，policy）需要开始产生动作：$a_{1}\sim \pi_{\theta}(\cdot|s_{1})$，由此得到一个奖励$R(s_{1},a_{1})$，并转移状态到$s_{2}=[s_{1},a_{1}]$（此处$[\cdot,\cdot]$表示把两个字符串给拼起来。重复上述过程，直至这一幕（episode）结束，或者达到了最大的生成步数，又或者生成了$$ token
最终，我们得到了一个trajectory，记作$\tau ={ a_{1},a_{2},\cdots,a_{H} }$

2.2. Test-Time Scaling Method

这里是在规定，我们要考察哪些TTS方法。主要是Best-of-N（BoN），beam search以及Diverse Verifier Tree Search（DVTS）
这里需要注意一个点，因为lookahead search效率不高，所以这里不对这种方式进行评估，且涉及到lookahead search的方式也都不评估（例如Monte Carlo Tree Search，MCTS）

2.3. Compute-Optimal Test-Time Scaling

感觉这里实质上就是给出一个优化函数，即，我们希望学习参数，使得回答正确率尽可能高（可这不是废话吗？还是说，这里是想告诉我们，推理模型的训练，不是一般的那种llm的训练了，是得关注问题准确率，而非一定要重复标准答案？）

问题：什么是lookahead search？
什么是Compute-Optimal Test-Time Scaling？
