例1. LeNet的搭建

知道各个基本的层，例如Conv2d的计算过程，Avg Pooling、Max Pooling的计算过程

此外，需要掌握编写神经网络的类的细节：

第一，网络的类一般是继承了nn.Module类的

第二，因为继承了这个类，所以也需要先初始化一下，也就是super(Your_Network_Name, self).__ init__()

第三，一般我们把网络里可训练的层的初始化放到__ init__函数里

了解常见层的计算过程（这里主要介绍的是CNN里常用的卷积和池化），能根据输入数据的形状以及各个层的参数，算出输出数据的尺寸

例2. GoogLeNet的搭建

例3. ResNet的搭建

首先，ResNet的搭建基本就是靠堆叠残差块实现的。残差块分了两种模式，18和34层的ResNet，用的是BasicBlock，而50及更高层数的ResNet用的是Bottleneck

ResNet的实例化，一般就是通过指定每种残差块的个数来实现的。一般它会被分成四个“大的”layer，每个layer里可能有若干个残差块，我们只需要通过一个列表，指定这四个“大”layer里残差块的格式，即可得到具体的ResNet

2.定制数据集

这里介绍了一般的关于现有数据集的一些操作，以及怎么定制一个自己的数据集

2.1. 使用现有数据集

直接当函数调用即可。例如，要调用CIFAR10的训练数据，则执行形如这样的代码即可trainset = torchvision.datasets.CIFAR10(root='./data', train=True, download=True, transform=transform)
注意到这里有transform参数，它代表了我们要对数据进行的预处理的操作。传进去的一般是transforms库里的Compose得到的一个一系列的操作，例如：transform = transforms.Compose([transforms.ToTensor(), transforms.Normalize((0.5, 0.5, 0.5), (0.5, 0.5, 0.5))])（这里可能还需要我们了解常见的数据预处理的操作。例如，此处的ToTensor就是把numpy格式的图像数据给转化成torch格式的图像数据，二者的主要区别在于Channel维度在第几个维度，numpy格式是\(H\times W\times C\)，而torch格式是\(C\times H\times W\)

上面我们得到了现有的数据集，但实际使用时，需要把数据放到一个dataloader里，以便加载使用。一般用torch.utils.data.DataLoader，例如：trainloader = torch.utils.data.DataLoader(trainset, batch_size=4, shuffle=True, num_workers)，这里我们可以通过shuffle参数指定是否打乱数据集，以及可以用multiprocessing workers并行地加载数据

2.2. 定制自己的数据集

2.2.1. 定制自己的数据集，首先得有自己的数据。然而原始的数据可能格式不太对，需要一些转换，这时可能要用到一些库了

2.2.2. 下一步是打标签。这里是以经典的图像分类任务为例，那可以考虑labelimg这个工具。当然还有其它的，例如labelme

2.2.3. 有了数据以后，类似上面调用现有数据集的方式，我们还是需要一个数据集的类，从而可以通过这个类来获取数据集。因为本质上，我们最后希望的效果是通过dataloader来获取batch data，而dataloader获取batch data的本质是，生成索引，然后每个batch就用相应的batch索引，从数据集类里获取数据（这也是为什么实例化一个dataloader的时候我们要传入dataset）
我们要继承的是一个torch.utils.data.Dataset类，它代表了一个数据集的抽象类。我们自定义的时候，重点关注三个函数：__ len__函数，它可以返回数据集的大小；
__ getitem__函数，它支持根据索引取出具体的数据，例如dataset[i]就可以取出第i个样本。这种方式的好处是，我们不需要每次都把所有的数据加载到内存里（毕竟这样一旦数据集大点，可能内存就炸了），而是要用到哪些数据时，我就加载这些数据
__ init__函数，一个类当然要有初始的函数了。具体到这里的例子，我们可能会在这里放一些类似于数据路径、要对数据进行的预处理操作transform等，以供__ getitem__或其它函数使用

2.2.4. 实际使用时，我们可能需要定制化的transform，此时则可以考虑把操作给写成一个类，然后transforms.Compose的时候，我们就传入这些自己写的类就行（当然，此时需要修改它的__ init__方法和__ call__方法。以及，它继承的是object类）

综上，我们定制了自己的数据集的类，它可以顺利加载数据，且进行相应的数据增强操作（如果有），并且想定制数据增强操作也是可以的。之后就是按照一般流程走了，实例化一个数据集的类，再构造对应的dataloader即可

3.使用服务器

3.1. 简介：

服务器，一般都是linux系统。且因为linux系统中，一切都是文件，而配置一个服务，本质上就是在修改配置文件里的一些设置；所以，日常使用时一定会编写文档，因此需要一个文档编辑器。最经典的就是vim了

再简单介绍一下vim。它有三种模式，输入模式、命令模式和末行模式

输入模式，就是一般的模式，我们打字，就能编辑文档。一般是通过a、i、o等键位（例如最常用的，i表示换到插入模式），从命令模式转到输入模式；按Esc键，则可以从输入模式切换到命令模式。从命令模式，按冒号可以切换到末行模式（也因此，一些常用的退出、保存等操作，都是带个冒号的。比如，:q就是退出，:w就是保存，:wq就是强制保存退出）；而从末行模式，再按Esc，则可以切换到命令模式（由此可以看出来，Esc键是可以在任何情况下切到命令模式的。事实上，命令模式也充当了一个中介的作用，毕竟输入模式和末行模式相互之间无法直接切换，得通过命令模式来，所以切换到命令模式当然得尽量快捷简便）

4.训练过程可视化

这里介绍的是用TensorBoard进行可视化（它相比于老式的一大堆print，可视化效果更好，也更加方便些）（不过这并不意味着print的方式就被彻底淘汰了）

使用TensorBoard的五个步骤：

1. 记录想要观察的张量。例如，w2_hist = tf.summary.histogram("weights2", W2)；cost_summ = tf.summary.scalar("cost", cost)

2. 合并所有的summaries（简单理解，就是把所有的记录合并一下）。summary = tf.summary.merge_all()

3. 创建writer，并添加graph

这里应该是要把结果给保存到某个路径下，因此需要用到一个writer；以及，需要添加graph（这里添加graph，主要是为了得到模型的架构，以方便后续的可视化；还有用于分析模型的复杂度等）
writer = tf.summayr.FileWriter('./logs')
writer.add_graph(sess.graph)

4. Run summary merge and add_summary（但这一步好像是tensorflow 1.x里的东西了，在2.x的版本里被简化了？所以感觉不会考代码啊）

5. Launch TensorBoard

这个倒是常见，tensorboard --log-dir=./logs，指定一下记录的文件的路径，就可以打开tensorboard了
