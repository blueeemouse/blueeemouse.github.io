最小栈

Posted on 2025-05-22 In algo , 算法面试题

原题链接在这

题目描述：

设计一个支持 push ，pop ，top 操作，并能在常数时间内检索到最小元素的栈。

实现 MinStack 类:

- MinStack() 初始化堆栈对象。

- void push(int val) 将元素val推入堆栈。

- void pop() 删除堆栈顶部的元素。

- int top() 获取堆栈顶部的元素。

- int getMin() 获取堆栈中的最小元素。

示例 1:

输入：

["MinStack","push","push","push","getMin","pop","top","getMin"]

[[],[-2],[0],[-3],[],[],[],[]]

输出：

[null,null,null,null,-3,null,0,-2]

解释：

MinStack minStack = new MinStack();

minStack.push(-2);

minStack.push(0);

minStack.push(-3);

minStack.getMin(); –> 返回 -3.

minStack.pop();

minStack.top(); –> 返回 0.

minStack.getMin(); –> 返回 -2.

提示：

- -2^31 <= val <= 2^31 - 1

- pop、top 和 getMin 操作总是在 非空栈 上调用

- push, pop, top, and getMin最多被调用 3 * 10^4 次

思路：

如果只是要实现一个栈，那还是比较简单的，用数组就可以进行模拟了。这道题多出了一个要求，就是要能用常数时间找出栈里的最小元素。因为要用常数时间，所以显然不能现排序，应该维护某个东西，可以直接查到当前栈的最小元素
提示部分，给出val的范围，说明可以用int就表示；pop、同配、和getMin都是在非空栈上调用，所以不需要考虑一些不合法的操作，全都当成合法的就行；push、pop、top和getMin最多各被调用$3 * 10^4$次，这里面最重要的可能是：push最多只会被调用$3 * 10^{4}$次这个信息，因为这说明了栈最大的时候也就是$3 * 10^{4}$的长度。所以我们用数组模拟的时候，可以开一些辅助空间（如果有需要），而相对不太需要担心超出限制

回到题目本身，一开始的思路是，维护一个min_idx变量，它指示的就是栈里的最小元素的索引，这样我们要返回最小元素的时候，return e[min_idx]即可
但这样有个问题，就是不好维护。栈的元素里的变化主要是由push和pop两个操作引起的，如果发生了push，那其实维护min_idx是比较简单的，只需要看一下新入栈的这个元素和之前的最小元素的关系是怎样的，看看是否要更新min_idx即可；
但如果发生了pop，那维护起来就很麻烦了。因为，
如果pop出去的元素，不是最小元素，那没什么；
如果pop出去的元素是最小元素，就比较头疼了：如果pop出去的元素就是当前栈里的最小元素，且是唯一的，那么pop完，剩下的栈里的最小元素肯定就变化了，那就得重新找了，但如果真的此时重新找最小元素，显然就违背了题目里说的“用常数时间找出最小元素”；如果pop出去的元素确实是最小的，但不唯一，那前面确实还有最小元素，但我们又没记录它的索引，还是很麻烦，如果不找出它的索引，后续就不好维护；如果找出它的索引，显然违背了题目的要求……

这时可能会想，那如果我维护两个索引呢？一个是最小元素的索引，一个是第二小元素的索引？但可以想见，会非常麻烦，而且更重要的是，这种维护索引变量的方法，在遇到有多个相同的最小元素的时候会很麻烦。所以要维护的变量应该更加直接才行

这里可以考虑，我们开一个一样大小的辅助的栈，它的第i个元素，就是从0到i的这个栈里的最小元素。这样，比如当前的top指针是5，也就是栈顶是5，那么我们要求此时的栈的最小元素，也就return一下这个辅助栈的索引为5的元素即可
那如何维护呢？push的时候，考虑一下一般的情况，如果栈里已经有一些元素了，且假设辅助栈里前面的元素也已经好了；现在push进来一个新的元素，栈的大小增加1，我们的辅助栈也需要多加一个元素了。此时，只需要比较一下新加入的元素和之前的最小元素的大小即可，也就是比较一下插入的val和此前的最小元素min_stack[top_idx - 1]（这里，我们把辅助栈记成min_stack，top_idx表示栈顶指针），从而可以得到min_stack[top_idx]。如果是一开始，什么元素都还没有的时候呢？此时push进来一个元素，栈里也就一个元素，理所当然的，最小元素也是它，就这么更新就行
那如果发生了pop操作呢？此时维护起来也非常方便，体现出了维护一个min_stack数组的好处了。其实我们什么都不用做。因为，pop掉以后，top_idx自然会减1，之后如果我们要查询最小元素，还是直接返回min_stack[top_idx]即可，因为min_stack的定义就是这个意思：min_stack[i]，它记录的是，从0到当前索引i的栈（或者说是数组）里的最小元素。刚才pop掉的元素，它的索引显然是i+1，它当然不影响我的min_stack[i]了
