存在重复元素

Posted on 2025-05-15 In algo , 算法面试题

原题链接在这

题目描述：

给你一个整数数组 nums 。如果任一值在数组中出现 至少两次 ，返回 true ；如果数组中每个元素互不相同，返回 false 。

示例 1：

输入：nums = [1,2,3,1]

输出：true

解释：

元素 1 在下标 0 和 3 出现。

示例 2：

输入：nums = [1,2,3,4]

输出：false

解释：

所有元素都不同。

示例 3：

输入：nums = [1,1,1,3,3,4,3,2,4,2]

输出：true

提示：

- 1 <= nums.length <= 10^5

- -10^9 <= nums[i] <= 10^9

思路：

第一眼看过去，觉得和之前的某道题特别像，因为也有类似的判断重复元素的存在这个步骤，其实是这道题[[字符串中的第一个唯一字符]]，但那道题的思路不能直接拿过来用，因为有个很关键的点在于，那道题里，字符串的元素只可能是26个小写字母之一，我只需要开一个数组就可以表征所有的可能的字符，以及用它来记录每种字符出现的次数。而这道题给的是一个数组，数组里的数取值范围是很大的，如果还用开数组这种方法来记录每种元素出现的次数，那空间开销太大了（进一步想想，其实这道题还可以改成连续区间吧，那更是不可能用数组来记录了）

其次，倒也容易想到哈希表那一套，但主要是不太熟……不过哈希表它大概是可以用来快速判断某个元素是否存在的，这是比较基础的。基于这一点，容易想到：遍历一遍数组，并不断判断当前在看的这个元素是否在集合里出现过，出现过说明就有重复元素了，直接return true；没出现过就把它加到集合里，然后继续遍历。cpp的话，unordered_set是哈希表的一个应用，它的底层是哈希表，而更上层也提供了一些接口，例如insert()，find()

代码：

这里用到了unordered_set的一个insert方法，它返回的是一个pair<iterator, bool>
其中，pair.first是一个指向新插入元素的迭代器（如果插入成功了）或者是一个指向已经存在的相同元素的迭代器（此时就是集合里已经存在这个元素了）
pair.second则是bool类型的变量，表示是否插入成功，true的话就表明插入的这个元素在此之前不在集合里，此时已经成功插入；false的话就表示这个元素之前已经被插入到集合里了，这一次插入失败了（因为不用插入相当于没有插入了）
