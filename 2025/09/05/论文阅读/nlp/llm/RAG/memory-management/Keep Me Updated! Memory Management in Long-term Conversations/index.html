Keep Me Updated! Memory Management in Long-term Conversations

Posted on 2025-09-05 In 论文阅读 , nlp , llm , RAG , memory management

Venue：EMNLP 22 Findings

date：2022-10-17

动机：

论文关注长期对话中的动态记忆管理问题。具体来说，我们与模型交流时，随着内容的增多，历史信息也不断增多。而后续的谈话有可能需要再次利用历史信息，且新的对话内容可能和历史信息发生冲突，导致历史信息需要更新（例如，新的对话里表明已经不生病了，则原来历史信息里保留的生病的信息就可以丢弃了）
所以总结一下，就是，如何让对话系统在多轮对话中跟踪、更新用户最新信息，管理动态记忆，实现更加自然连贯的长期对话

简介：

论文针对问题，首先明确定义了这种新的任务：在多会话对话中，每次给定上下文以及历史记忆，需要先给出当前的合理的回答；回答结束后，需要根据这一次的对话，更新历史记忆，以用于下一轮对话

方法上，论文专门构建了一套多会话数据集，同时设计了一套系统，包含三个主要部件：

Memory Grounded Response Generation Response Generation：利用记忆，检索相关内容，进行合理回复（这一部分很类似naive RAG，只不过此处的检索库是历史记忆而已。我们还是检索top-k的词条，用于辅助回答）

Dialogue Summarization：将当前会话机型抽象总结，提炼出关键信息，用于更新历史记忆（此处会专门训练一个summarizer模型，同时会有手工标注的“对话历史--总结句子集合”这样的数据对，用于训练summarizer

Memory Update：事先定义了四种操作（根据新内容和记忆之间可能发生的关系定义的），结合旧记忆和当前对话的抽象信息进行更新（此处因为实现定义了四种操作，是根据历史记忆和当前对话摘要之间的关系得到的，所以有一个四分类模型，它的输入就是\((m_{i},s_{j})\)，其中\(m_{i}\)表示历史记忆中的第i条，\(s_{j}\)表示当前对话摘要的第j条，输出则是四种操作之一的类别。遍历所有的记忆和对话摘要组合，确定哪些是要保留的，哪些是要剔除的，完成历史记忆的动态更新）

insight：

感觉看下来，没有哪个地方说得上是特别令人惊艳的；大概它切入点是比较新颖的，长对话的历史记忆管理

用非结构化文本来进行记忆管理，这个点，怎么说，或许可以是一个在其它领域借鉴的创新点？……还是感觉有点强行了，不应该这样硬凑创新点。说到底，如果在某个领域，我们希望用非结构化文本，方便人类理解，那么这就是自然的，也应该是我们的目标。如果没有这个需求，那不一定非要非结构化文本，结构化和非结构化文本，哪个效果好就用哪个吧

update操作被涵盖成四种预先定义的操作，然后具体的实现用一个单独训练的分类模型实现，这一套算是一个比较常见的trick了。针对这种比较灵活的问题，如果暂时没有特别好的处理方法，那么用这种相对机械、工程的做法也不是不行吧（权宜之计）（且一般有可能需要自己构造数据。毕竟，要训练这种模型，就得有“输入数据--对应操作”这样的数据对，然而你的操作一般是自己定义的，所以基本找不到现成的数据集）

contribution：

提出一套记忆管理机制，且是用非结构化文本，利于人类理解

提出第一个韩语的长期（这里指的是有多个会话，会话之间有时间间隔这样的吧）对话数据集，可用于记忆管理方面的研究
