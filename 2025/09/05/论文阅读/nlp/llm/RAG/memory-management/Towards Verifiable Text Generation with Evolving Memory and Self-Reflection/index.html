Towards Verifiable Text Generation with Evolving Memory and Self-Reflection

Posted on 2025-09-05 In 论文阅读 , nlp , llm , RAG , memory management

Venue：EMNLP 24

date：2024-10-18

简介：

所谓verifiable text generation，是指，llm生成的时候，每一条内容（或者说主张，claim），都有附加的引用文档，从而使得生成内容具备可验证性。这是缓解幻觉的一大手段（本质上，这是将“生成内容的准确性”从不可直接观测的“模型参数知识“转移到了可以直接验证的”外部证据“）

论文提到的三大挑战，其实对应了它提出方法的三个阶段。第一个挑战，是verifiable text generation可能发生焦点转移现象（即the focus of the content changes over time）。因为它一般会涉及到长文本，而长文本可能不同的地方讨论的重点是不一样的。如果我们在重点切换之后，依然沿用前一个重点对应的检索文档池（也就是说，我们的检索文档池是静态的），那有可能压根找不到支持当前主张的证据（这也就是第二个挑战，即主张-证据之间的对齐不精准。如果证据不能支持主张，那就失去一开始的意义了，因为这样就无法避免幻觉了
最后一个挑战是检索的广度和精度的问题。一方面，如果检索的文档过多，有可能判断失误，会把一些其实并不支持主张的证据也加到引用里（也就是受到噪声的影响了）；但本身，如果检索的广度不够，有可能就找不到需要的支持证据。所以二者之间需要一个权衡

动机：

论文是想解决上述提到的三个挑战

为此，它提出的方法包含了三个阶段，分别用于解决一个挑战
第一个阶段，是generation evolving memory。这里，论文构建了长时记忆和短时记忆两个模块，其中短时记忆是会随着过程动态更新的，因此当焦点转移的时候，如果哦我们发现长时记忆和短时记忆里都找不到支持主张的证据，就重新检索，并把检索到的支持主张的证据加入到短期记忆里
第二个阶段，是citation construction。此阶段主要是解决主张-证据对齐不精准的问题。具体来说，我们对一个主张的对应证据，会用NLI模型进行检查，如果发现它是支持该主张的，则通过，并进行化简；如果发现它是不支持该主张的，则说明这个证据找的不太对，应该再继续找。因此会在长时记忆和短时记忆里重新检索：若发现长短时记忆力是存在支持证据的，则进行检索；否则说明，当前的记忆库里是没有支持证据的，需要重新找，则进入到第三阶段
第三个阶段，我们可以发现，它不是必须的。如果已经找到了支持主张的证据，就不需要这一步。如果一直没找到，则我们需要重新检索。这里会利用llm生成多样化的query，从而进行多样的检索，看看是否能检索到支持主张的证据。如果迭代若干次一直找不到，则不再浪费时间处理这个主张，直接把最后一次的证据附上（尽管可能它是不支持该主张的）

insight：

长短时记忆看起来稍微精妙一点。本质上是把存储的内容分两套，发现证据不够用的时候就检索，然后对短时记忆进行更新（当然长时记忆也有相应的更新）

后两个阶段，感觉没有太多精妙的。不过其中有一步，是让llm生成多样化的query，用于检索。这其实本质上就是rewrite query，然后多次进行检索，看能不能检索到不一样的东西。很常见的操作了，也有论文在研究RAG中的query refinement。感觉这个方法虽然很笨，可是是有一点道理的。毕竟我们仅用原始问题进行检索，那么本质就是在用原始问题的embedding进行检索，这样检索到的只能是字面意义上比较相关的文档。然而，embedding应该不太能支持这么复杂的推理（再次cue到最近出的那篇Google deepmind讲embedding局限性的论文，感觉那篇确实有点直指要害了），所以需要我们通过改写query，把进一步的推理摆到台面上（或者说是推理显式化），再让模型去检索文档，才有可能弥补空缺

contribution：

提出一套VTG框架，改善了verifiable text generation的效果（其实就是改善了一开始提到的三个挑战）
