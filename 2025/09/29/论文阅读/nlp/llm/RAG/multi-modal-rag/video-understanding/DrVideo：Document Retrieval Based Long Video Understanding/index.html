简介：

一、研究的核心问题

现有长视频理解技术难以应对时长从几分钟到 1 小时以上的视频，核心需解决两大关键挑战，同时突破现有方法的技术瓶颈： 1. 长视频固有的技术难题 - 关键信息定位难：长视频（如 44 分钟的教学视频、10 分钟的电影片段）包含海量帧，现有方法难以精准筛选与查询相关的关键帧（如 “视频中使用的釉料颜色” 这类细节查询，易被冗余帧干扰）； - 长程推理能力弱：需整合视频中跨时段的事件逻辑（如 “主角从准备工具到完成手工的步骤衔接”），但现有模型受限于上下文窗口或模态融合能力，无法有效建立长时序关联。 2. 现有方法的应用局限 - 无法处理超长时间视频（如超过 1 小时的视频），多数方法仅针对几十秒的短视频设计； - 对开放域查询的适应性差，难以应对需要细节理解（如视觉特征、步骤细节）的复杂查询（如 “视频中人物操作工具的具体角度”）。 ## 二、现有方法的不足 论文明确指出两类主流长视频理解方法的核心缺陷，具体如下： ### 1. 视频 - 语言模型（Video-LMs）的局限

这类方法将视频编码为视觉 token 与语言 token 拼接后输入 LLM，但存在三大问题： - 输入长度受限：视觉编码器（如 CLIP-L-14）每帧输出 24×24 个 token，而主流 Video-LM（如 LLaVA-NeXT-Video）的最大上下文窗口仅 8192 个 token，无法处理全帧（如 10 分钟视频按 0.5FPS 采样约 300 帧，总 token 数远超窗口限制）； - 帧采样盲目：采用固定大步长均匀 / 随机采样（如 PLLaVA 用 16 帧步长、IG-VLM 用 6 帧步长），未考虑内容相关性，易丢失关键帧（如 “蛋糕烘焙中的烤箱温度调整” 这类短时段关键操作）； - 视觉信息定位难：视觉 token 与语言 token 简单拼接后，LLM 难以在长序列中定位与查询相关的视觉信息（如查询 “视频中人物穿的衣服颜色”，模型易被背景帧的视觉 token 干扰）。 ### 2. 基于 LLM 的视频字幕转换方法的局限 这类方法（如 LLoVi、VideoAgent）将视频拆分为短片段并转为文本描述，再利用 LLM 推理，但存在两大关键缺陷： - 关键帧定位不准确：依赖 LLM 先验知识推断缺失信息，再通过 CLIP 图像 - 文本相似度筛选帧，缺乏对视频内容的全局把控。例如 VideoAgent 仅采样 5 帧生成文本，易因初始帧代表性不足导致后续定位偏差（如 “识别手工制作步骤时，漏选‘打结固定’的关键帧”）； - 信息丢失严重：将视频转为字幕或简短描述时，会丢弃视觉细节。例如描述 “女性在镜子前” 无法覆盖 “衣服款式、颜色” 等关键信息，导致 LLM 回答 “女性穿什么衣服” 时只能盲目猜测，无法提供准确依据。 ## 三、论文提出的解决方法 论文提出DrVideo 框架，核心思路是将 “长视频理解” 转化为 “长文档检索与理解”，通过五大模块协同解决上述问题，具体流程如图 1 所示： ### 1. 视频 - 文档转换模块：建立长视频的文本化表示 - 操作逻辑：用预训练视觉 - 语言字幕模型（如 LaViLa、LLaVA-NeXT）将视频帧（或短片段）转为简短文本描述(\(S_{V_t}\))，构建初始视频文档\((Doc_{init}=\{\{1,S_{V_1}\},\{2,S_{V_2}\},...,\{T,S_{V_T}\}\})\)（T为视频总帧数）。例如将 “揉面团” 的帧转为 “人物用手掌按压面团，表面逐渐光滑” 的文本，让 LLM 能通过文本快速把握视频内容； - 关键设计：支持多字幕模型替换（如 BLIP-2），并针对不同数据集优化采样率（EgoSchema 用 0.5FPS、Video-MME 用 0.2FPS），在减少冗余的同时保留关键信息。 ### 2. 文档（帧）检索模块：精准筛选初始关键帧 - 操作逻辑：用 OpenAI 嵌入模型生成初始文档中每个帧描述的向量表示(\(E_{doc_t}\))，计算查询向量(\(E_{RT}\))与(\(E_{doc_t}\))的余弦相似度，检索 Top-K（默认 K=5）与查询最相关的帧（如查询 “釉料颜色” 时，筛选出包含 “粉色、绿色、蓝色釉料” 描述的帧）；（这里实际实验的时候，是会进行一个均匀采样的） - 作用：快速缩小关键帧范围，避免后续处理全量帧的高计算成本，同时为后续信息增强提供基础。 ### 3. 文档增强模块：补充关键帧的细节信息 - 操作逻辑：对检索到的 Top-K 帧，用 LLaVA-NeXT 结合不同增强提示（AP）生成详细描述(\(L_{V_{t'}}\))，更新文档为({AD})（仅增强关键帧，非关键帧保留原描述）。例如针对 “人物操作工具” 的帧，提示模型生成 “工具为银色剪刀，刀刃与材料夹角约 45 度” 的细节描述； - 关键设计：增强提示分两类 ——A 类（图像详细描述）补充视觉细节，B 类（视觉问答）直接关联查询需求（如查询 “步骤顺序” 时，生成 “该帧为第三步，需在第二步搅拌后进行”），确保信息针对性。（都是在生成更加丰富的文字描述，本意是想深入挖掘这些检索回来的关键帧。看起来会损失很多的视觉信息，但是好在这里有B类增强提示，它会更加好地利用视觉信息，通过一个Visual QA，针对query来深挖关键帧的视觉信息） ### 4. 多阶段智能体交互循环：动态补全缺失信息 设计 “规划智能体（Planning Agent）+ 交互智能体（Interaction Agent）” 的迭代 loop，解决 “初始关键帧信息不足” 的问题： - 规划智能体：输入当前更新文档(i)、查询 Q 与历史分析记录 H，判断信息是否足够生成答案。若不足（如 “未找到釉料颜色的所有描述”），输出缺失原因并更新 H； - 交互智能体：基于 H 从非 Top-K 帧中筛选 N 个（N<K）缺失关键帧（如补充 “蓝色釉料使用” 的帧），并指定增强类型（A/B 类），调用文档增强模块更新文档； - 迭代终止条件：直至规划智能体确认信息充足或达到最大迭代次数（实验中默认 2 轮达最优性能）。（这个智能体的交互循环其实才是这套方法有效的根本原因吧，不然仅仅是前面的那些步骤，其实和现有方法感觉也没多大区别。但正是因为有这个智能体的交互循环，所以能减少关键帧的遗漏，尽可能减少视觉信息的损失） ### 5. 回答模块：基于链推理（CoT）生成答案 - 操作逻辑：将最终文档({final})与查询输入 LLM，采用 CoT 策略生成答案、置信度与推理过程。例如回答 “釉料颜色” 时，先梳理 “帧 15 提到粉色、帧 32 提到绿色、帧 46 提到蓝色”，再汇总得出 “3 种颜色” 的结论； - 优势：提升答案准确性的同时，通过推理步骤确保可解释性（如追踪 “为何排除其他颜色” 的逻辑）。 ## 四、方法的创新性 1. 任务范式创新：首次将长视频理解转化为长文档任务突破传统 “视频→视觉 token” 的处理思路，通过文本化将长视频转化为 LLM 擅长的长文档格式，直接复用 LLM 在关键信息定位（如文档检索）与长程推理（如跨段落逻辑分析）的优势。例如处理 44 分钟的 Video-MME 视频时，文档化后 LLM 可像分析长文本一样梳理 “从材料准备到成品展示” 的全流程，无需处理复杂的视觉 token 序列。 2. 动态信息补全：多智能体循环实现精准信息挖掘 - 区别于 VideoAgent 等方法 “一次性采样 + 固定增强” 的模式，设计双智能体迭代机制：规划智能体负责 “判断信息缺口”（如 “是否遗漏釉料颜色”），交互智能体负责 “精准补全”（如补充未检索到的蓝色釉料帧），实现 “需求驱动→信息补全→验证优化” 的闭环； - 实验表明，该机制在 EgoSchema 数据集上仅需 2 轮迭代即达最优性能，且关键帧定位精度比 VideoAgent 高 6.2%。 3. 信息增强的适应性：按需匹配增强类型 针对不同关键帧的信息需求，自适应选择增强类型（A 类补充视觉细节、B 类关联查询），避免单一增强导致的信息冗余或不足。例如： - 对 “人物动作” 类帧用 A 类增强（如 “手部握笔姿势”），补充视觉特征； - 对 “步骤顺序” 类帧用 B 类增强（如 “该步骤在搅拌之后”），直接关联查询逻辑； - ablation 实验证明，同时使用两类增强的准确率（62.6%）显著高于单独使用 A 类（60.4%）或 B 类（61.8%）。 4. 高效性与通用性：零训练成本 + 宽场景适配 - 零训练设计：基于预训练 VLM（如 LaViLa）与 LLM（如 GPT-3.5/4）构建，无需额外训练，在 RTX 4090 上即可复现结果，降低研究门槛； - 跨场景适配：在 3 分钟的 EgoSchema、10 分钟的 MovieChat-1K、44 分钟的 Video-MME 三大基准数据集上均显著优于 SOTA，尤其在 Video-MME 中结合字幕达 71.7% 准确率，超过 Gemini 1.5 Flash（68.8%）、GPT-4V（56.9%）等 proprietary 模型。
