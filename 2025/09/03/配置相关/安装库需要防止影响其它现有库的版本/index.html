安装库需要防止影响其它现有库的版本

Posted on 2025-09-03 In 配置相关

如题，在配置环境，需要加装一些库的时候，如果简单的pip install，可能会影响到我们现有的库，最后可能出现：新装的库没问题了，但原来没问题的库反而报错了。为此，可以考虑pip install的时候，加上--no-deps。这个是来告诉pip，“不要安装或升级这个包声明的依赖”，只装我点名的包的本体，从而防止把已经配好的库的版本打乱了（当然，一些很复杂的情况下，可能新的库和旧库的当前版本会出现冲突。这就要视情况而定了）

另外，在jupyter notebook里配置环境的时候，如果要在单元格里执行终端命令，那么需要加上“shell魔法”，也就是在指令前面加上一个!
还可以考虑跟一个"{sys.executable}"，它是用来确保pip安装的包，是安装到当前正在运行的这个环境里，而不是系统的其它python里（不过如果是正常在终端执行的话，事先激活要安装到的那个环境，应该就行了）
另外，有时候安装的时候会带个-m，它的意思是，让这个 Python 去运行某个模块当作脚本。python -m pip 就是“用这个 Python 调用它自带的 pip”，避免出现 python 和 pip 指向不同环境的问题。
那么一个完整的，在notebook的code格里安装库的命令，类似这样：
!"{sys.executable}" -m pip install torchprofile --no-deps
