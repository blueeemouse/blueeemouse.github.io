目标和

Posted on 2025-08-19 In algo , 动态规划 , 背包 , 0-1背包

原题链接在这

题目描述：

给你一个非负整数数组 nums 和一个整数 target 。

向数组中的每个整数前添加 '+' 或 '-' ，然后串联起所有整数，可以构造一个 表达式 ：

  • 例如，nums = [2, 1] ，可以在 2 之前添加 '+' ，在 1 之前添加 '-' ，然后串联起来得到表达式 "+2-1" 。

    返回可以通过上述方法构造的、运算结果等于 target 的不同 表达式 的数目。

    示例 1：

    输入：nums = [1,1,1,1,1], target = 3

    输出：5

    解释：一共有 5 种方法让最终目标和为 3 。

    -1 + 1 + 1 + 1 + 1 = 3

    +1 - 1 + 1 + 1 + 1 = 3

    +1 + 1 - 1 + 1 + 1 = 3

    +1 + 1 + 1 - 1 + 1 = 3

    +1 + 1 + 1 + 1 - 1 = 3

    示例 2：

    输入：nums = [1], target = 1

    输出：1

    提示：

  • 1 <= nums.length <= 20

  • 0 <= nums[i] <= 1000

  • 0 <= sum(nums[i]) <= 1000

  • -1000 <= target <= 1000

    思路：

    这个也是比较简单、典型的背包问题。我们需要给每个元素都选择添加一个正号或者负号，不妨记添加了正号的那些元素之和为a，添加了符号的那些元素，它们的绝对值之和（也就是本身的元素和，先不考虑添加的负号。题目已经说了，数组里的元素都是非负数）为b，那么根据题意，应该有a-b=target，且默认的有a+b=sum(nums)
    因此，应该有\(a = \frac{target+sum(nums)}{2}\)，\(b=\frac{sum(nums)-target}{2}\)
    故，我们可以把问题转化成：有多少种方案，可以从nums中挑出元素，使得挑出的元素之和为a或为b。当然这是大体的思路，细节上还有一些要注意的。例如，当我们判断target加上/减去sum(nums)为奇数，则此时a和b均为小数，但nums里的元素都是非负数，显然加不出来小数，所以可以直接return 0；且因为a和b必然都是非负的，所以sum(nums)必须大于等于target，否则也可以直接return 0。如果这些特殊情况都没有发生，那么正常做即可
