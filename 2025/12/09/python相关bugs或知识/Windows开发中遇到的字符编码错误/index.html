Windows开发中遇到的字符编码错误

Posted on 2025-12-09 Edited on 2025-12-14 In python相关bugs

错误信息形如'gbk' codec can't decode byte 0xa6 in position 5841: illegal multibyte sequence。它表示的含义是，（一般是在读取文件的时候）默认使用了 'gbk' 编码去解码，但这个文件实际上是 'utf-8' 编码保存的。当它遇到一个 'gbk' 无法理解的字节（0xa6）时，就立刻报错了

印象里好像是Windows里这种问题比较多，有点忘了Linux系统下有没有遇到了。但在中文 Windows 环境下，系统的默认代码页是 'gbk'（或 'cp936'）。很多 Python 程序如果没有明确指定编码，就会顺从地使用系统默认编码来读取文件

要解决的话，可以考虑以下几种方法：

  • 把文件转成UTF-8编码。也就是打开这个文件，选择以其它编码（utf-8）保存

  • 可以在终端里执行$env:PYTHONUTF8 = 1这个命令，之后再在这个终端里运行脚本的话，这个环境变量它会告诉python解释器：“无论系统默认是什么，所有文件 I/O 操作都强制使用 UTF-8 编码”。这样，在读取文件时，就用的是 UTF-8 去解码了

  • 当然，还可以在读取文件的时候就指定好编码方式（只不过如果是用别人的代码，或者是比较多的地方读取了，但是又没改，或许可以考虑在终端指定编码方式？感觉这样好像也没什么问题啊……
