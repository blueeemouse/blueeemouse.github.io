python环境定期清理

Posted on 2025-12-03 In python相关bugs

其实这不是一个bug，只是在清理c盘空间的时候，先在文件管理器里看到.cache文件夹下有几个比较大的库的文件夹，这些是对应的库的缓存（虽然不知道是对应到哪个环境啊）

顺手问了一下ai，发现其实可以考虑对python环境进行一些清理（比如各种的conda环境。这些其实很吃空间），比手动清理好些。毕竟不容易误删。据ai所说，可以考虑定期执行一下pip cache purge命令，试了一下发现直接清理出了大概20g的空间

这是因为，这个命令表示把全局的、所有的用户缓存都给清理掉（purge表示净化、清除，是很彻底的）

一般在以下场合可以考虑使用这个命令（不止是想清理空间的时候）：

  • C盘空间不足时：就像你这次遇到的情况，这是最直接有效的解决方法。

  • pip安装出现奇怪错误时：有时候，缓存的安装包可能损坏了，导致安装失败。这时可以尝试 pip cache purge 清理一下，让 pip 重新下载一个完好的文件。

  • 完成一个大项目后：如果知道短期内不会再安装那些特定的依赖，可以清理一下释放空间。
