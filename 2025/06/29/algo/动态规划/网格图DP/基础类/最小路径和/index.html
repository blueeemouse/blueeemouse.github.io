最小路径和

Posted on 2025-06-29 Edited on 2025-06-30 In algo , 动态规划 , 网格图DP , 基础类

原题链接在这

题目描述：

给定一个包含非负整数的 m x n 网格 grid ，请找出一条从左上角到右下角的路径，使得路径上的数字总和为最小。

示例 1：

说明：每次只能向下或者向右移动一步。

输入：grid = [[1,3,1],[1,5,1],[4,2,1]]

输出：7

解释：因为路径 1→3→1→1→1 的总和最小。

示例 2：

输入：grid = [[1,2,3],[4,5,6]]

输出：12

提示：

  • m == grid.length

  • n == grid[i].length

  • 1 <= m, n <= 200

  • 0 <= grid[i][j] <= 200

    思路：

    因为知道是动态规划类的题嘛，所以自然的会往这方面想。至于递推式，也很好想：\[min\_cost[i][j]=\min(min\_cost[i-1][j], min\_cost[i][j-1])+cost[i][j]\]其中i和j表示网格中某个位置的坐标。并且我们以左上角为原点往右下角走则i和j增大，且i表示行，j表示列。大体思路已经有了，剩下就是具体的实现了。类似一维的情况，如果我们用朴素的递归方法实现，那么时间复杂度就是\(O(2^{mn})\)（因为有mn个格子，在求每个格子的时候都要考虑上方和左方的min cost。这里姑且忽略边界上的点，因为这不会影响复杂度的级别）。改进的方法就是记忆化搜索，递归搜索过的位置我们会进行记录，不再进行搜索。这样就是\(O(mn)\)的时间复杂度了。空间复杂度也是\(O(mn)\)，因为需要开一个二维矩阵存储每个位置的最小开销
