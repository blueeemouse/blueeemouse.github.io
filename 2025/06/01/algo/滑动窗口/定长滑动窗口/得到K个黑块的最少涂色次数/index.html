得到K个黑块的最少涂色次数

Posted on 2025-06-01 In algo , 滑动窗口 , 定长滑动窗口

原题链接在这

题目描述：

给你一个长度为 n 下标从 0 开始的字符串 blocks ，blocks[i] 要么是 'W' 要么是 'B' ，表示第 i 块的颜色。字符 'W' 和 'B' 分别表示白色和黑色。

给你一个整数 k ，表示想要 连续 黑色块的数目。

每一次操作中，你可以选择一个白色块将它 涂成 黑色块。

请你返回至少出现 一次 连续 k 个黑色块的 最少 操作次数。

示例 1：

输入：blocks = "WBBWWBBWBW", k = 7

输出：3

解释：

一种得到 7 个连续黑色块的方法是把第 0 ，3 和 4 个块涂成黑色。

得到 blocks = "BBBBBBBWBW" 。

可以证明无法用少于 3 次操作得到 7 个连续的黑块。

所以我们返回 3 。

示例 2：

输入：blocks = "WBWBBBW", k = 2

输出：0

解释：

不需要任何操作，因为已经有 2 个连续的黑块。

所以我们返回 0 。

提示：

- n == blocks.length

- 1 <= n <= 100

- blocks[i] 要么是 'W' ，要么是 'B' 。

- 1 <= k <= n

思路：

一开始看到这题目，想得有点复杂了。因为，最简单的情况当然是原始字符串里已经有连续k个黑色块了，则直接返回0；
但如果不是这种情况，那么要判断的话，比如我们关注索引i开始的连续黑色块，它可能是一个长度为k-5的连续黑色块，那么此时我们就需要去看，索引i+(k-5-1)处后面的长度为5的地方里，有没有连续黑色块，以及有的话能否在通过改变中间的白色块后得到连续k个黑色块。这样，即使我们通过遍历一次字符串，维护了一个数组cnts（用于表征从索引i开始的连续黑色块长度。且在维护的过程中，我们就可以知道是否原字符串里有连续k个黑色块了，有的话直接return），之后我们还是要遍历一次这个cnts，并且得到当前索引出发的连续黑色块数量，我们还要往后看，来检查，这就很麻烦了

此时重新审视题目，发现，其实可以换个思路：如果我们还是用一个长度为k的滑动窗口来遍历字符串，则我们只需要看一下当前窗口里黑色块的数量是多少，如果就是k，则说明原字符串里已经有连续k个黑色块了，则直接return；如果不是k，那么，如果我们最后选择了要把这个滑动窗口里的所有字符都改成'B'，也就是把这个滑动窗口变成长度为k的连续黑色块，那么，我们应该做的操作次数就恰好等于这个滑动窗口里的白色块数量。此时容易想到，滑动窗口滑动的过程中，我们要动态维护窗口内的黑色块和白色块数量，但这是很容易的（因为滑动窗口只需要关注进出的那两个元素）。而滑动的过程中，我们只需要根据窗口里最少的白色块数量，来更新“最少操作次数”即可。事实上，最小操作次数就等于所有滑动窗口里最少的白色块数量（这里是统一的，包括了原始字符串就有连续k个黑色块的情况，因为此时一定有一个滑动窗口里是没有白色块的，也就是白色块数量=最小操作次数=0）
