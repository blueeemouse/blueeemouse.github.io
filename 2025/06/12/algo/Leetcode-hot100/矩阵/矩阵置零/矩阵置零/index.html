矩阵置零

Posted on 2025-06-12 Edited on 2025-06-19 In algo , Leetcode-hot100 , 矩阵

原题链接在这

题目描述：

给定一个 _m_ x _n_ 的矩阵，如果一个元素为 0 ，则将其所在行和列的所有元素都设为 0 。请使用 原地 算法。

示例1：

输入：matrix = [[1,1,1],[1,0,1],[1,1,1]]

输出：[[1,0,1],[0,0,0],[1,0,1]]

示例2：

输入：matrix = [[0,1,2,0],[3,4,5,2],[1,3,1,5]]

输出：[[0,0,0,0],[0,4,5,0],[0,3,1,0]]

提示：

- m == matrix.length

- n == matrix[0].length

- 1 <= m, n <= 200

- -2^31 <= matrix[i][j] <= 2^31 - 1

进阶：

- 一个直观的解决方案是使用  O(mn) 的额外空间，但这并不是一个好的解决方案。

- 一个简单的改进方案是使用 O(m+n) 的额外空间，但这仍然不是最好的解决方案。

- 你能想出一个仅使用常量空间的解决方案吗？

思路：

先不管什么就地不就地的，光看这个题意描述，直观上想，难点应该在于：当我们找到某个0，然后把它所在的行和列的元素都置为0时，会平白无故多出很多0，我们就分不出哪些是原来的，哪些是新产生的（只有原来的0才会产生新的0）。所以我们考虑做一些标记，以便能判断出到底哪些是原来的0
那其实我们可以先遍历一遍原矩阵，把所有原始的0的行列都存起来（可以不开一个矩阵，而是用两个向量来分别存储原始0的行和列）。遍历完，我们只需要再根据这些行号和列号，去单独把这些地方的元素给置为0即可。此时就不用担心新的0干扰判断了

这个方法感觉上是比较简单的，稍微分析一下它的时间复杂度：一开始需要遍历一遍完整矩阵，就已经是\(O(mn)\)了；之后要根据原始0的行号和列号来置0，最差的情况下，应该是\(O(mn*m)+O(mn*n)\)的复杂度了，平均复杂度也就是一半，还是这个量级的。所以总体的时间复杂度应该是\(O(m^{2}n+mn^{2})\)（感觉真挺复杂的，估计也是题目比较简单，m、n不超过200，所以能过）
至于空间复杂度，主要是开了两个vector来存原始0的行列号。最差和平均的情况下，应该都是\(O(mn)\)级别的空间。所以其实开销也很大。这个比较直观的做法还是不够好的

改进：

首先，我们看看进阶里的要求。直观的解决方法，应该也差不多就是上面那种方法，简单但开销大。第一个改进是把空间复杂度变成\(O(m+n)\)。这里，关注到m和n，就是行数和列数，所以它可能是在这里进行化简。事实上，从上面的思路也可以发现，它其实是有重复的操作的：举个例子，我们在同一行，如果遍历发现了两个0元素，那存行坐标的时候，其实只用存一个就好，因为这两个元素是同一行的嘛。但上面的做法就会存两个一样的行坐标，且后面还会遍历两次同一行，显然浪费时间。所以这里就可以进行改进（此时我们已经可以发现，空间复杂度降为\(O(m+n)\)了）。
考虑构建两个数组，一个存原始0的行坐标，一个存原始0的列坐标。不妨让存行坐标的就是m大小，存列坐标的就是n大小（当然这里m和n的上限都很小，直接构造两个大小为200的也不是不行，再大一点也可以，保险）。我们遍历原始矩阵元素的时候，还是正常操作：如果碰到一个原始0，就根据它的行号，在行坐标的那个数组里，找到对应索引处，记录一下这个行号里出现了一个0（所以存行坐标和存列坐标的两个数组，它的元素的含义就可以理解为是在这个行/列下出现的原始0的个数）
遍历完原始矩阵，我们就根据这两个数组来把对应的行和列的元素置为0就行。此时因为即使同一行/同一列有多个0，我们也只会遍历一次，所以，此时在把元素置为0的时候，最坏的和平均的时间复杂度也就是\(O(m*n)+O(n*m)=O(mn)\)了，比上面的时间复杂度要低了（代码就不单独放了，还挺简单的）
