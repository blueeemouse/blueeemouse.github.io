windows11配置mujoco环境

Posted on 2025-06-19 In 配置相关

因为rl大作业需要，所以要用一下mujoco仿真环境，为了方便，先试试在windows系统上能否正常配置（尽管新版本的mujoco那些，肯定是在linux系统上兼容的比较好的）

这里配置的是mujoco的mjpro150版本，一个比较老的版本，但据说这个版本在windows上兼容的比较好，考虑到仅仅为了作业，先试用一下，故安装此版本

其实现在的网上教程已经有很多了，但每个教程都很难完全覆盖每个人的问题。这里记录也只不过是记录一些我遇到的问题而已，因为每个人安装过程中遇到的问题未必一样，如果没有深入理解这些报错的原因，就只能尝试找找有没有其它人踩过坑，然后仿照它的解决方法

主体教程参考的是这个，不过在其中，我又遇到了一些问题：

1.在它创建虚拟环境，然后按照mujoco的配置文件安装库的时候，遇到一些问题。这个已经解决了，大概率是网络的问题，可以参考[[第一弹]]里的1.

2.在它提到进行mujoco-py测试的时候，我遇到了问题。具体来说就是，按照教程里运行那个测试脚本，但是报错如下：

看评论区，发现有人提到相关问题，有说是cython版本过高的，也有给出参考博客的（而参考博客里说是可能是因为Visual Studio、visual studio SDK没有和anaconda、python、pycharm安装在一个盘里，还是会报错的。也就是说，1.你需要这些东西都安装在一个盘里，2.还要确保安装的mujoco和mujoco_py都是150版本。但是我应该都装到C盘了，有点奇怪，感觉应该不是这个问题）正好这篇参考博客里的评论区也有人提到可能是cython版本过高的原因，于是尝试安装pip install cython==0.29.21（可能还要指定一下清华源或者别的源，不过大概率看网速了，这就一个库，应该也不是特别大，我是没指定就装好了）。重新装了cython，再运行主体参考的那篇博客里的测试mujoco的脚本，就成功了

3.在它提到要安装gym的时候，我没有按照它说的来，还是因为看评论区发现，最后如果按照它那个来装，似乎会有版本问题（虽然我没按照它的来实践一下，也不清楚是真是假），于是按照评论区的做法来，执行pip install gym==0.20 -i https://pypi.tuna.tsinghua.edu.cn/simple，成功安装后，再执行主体教程里最后的那个测试gym的脚本，是正常的。至此，配置应该算是结束了

貌似还有个隐患，但不知道后面会不会遇到。就是，现在安装的gym库似乎并不完整，因为我参考别的博客里的测试文件，发现总是会报错，少了什么什么module，然而它的测试文件里只导入了gym，再结合一下其中一段报错，很可能是gym没安装完整

似乎要安装完整gym，也只需要

或者按照上面的报错信息里的提示，进行'pip install -e .[all]' or 'pip install gym[all]'。但很担心随便搞容易把环境又给搞坏了，所以先用着，如果后面真的遇到了要用完整gym，再追加安装一下吧
