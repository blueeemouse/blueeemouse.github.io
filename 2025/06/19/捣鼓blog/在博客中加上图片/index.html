npm install hexo-renderer-marked
然后修改_config.yml文件，修改和增加内容：

post_asset_folder: true # 这是修改的

然后尝试了
这种形式添加图片，但是一来它在本地都看不了，二来在博客上似乎也不行

而传统的![]()的方法，相对路径的话，只要对了就可以显示，但部署上去之后不行；即使是给博客文件创建了对应的文件夹，然后把图片也放到里面去，此时用相对路径，也只能在本地看，博客上看不了；而如果换成绝对路径，好像本地都直接看不了了

之后尝试比较笨的方法，就是在source文件夹下创建一个专门存放图像的文件夹。此时用相对路径即可，但本地无法显示，而在博客上能显示（而且还有个我自己的问题，就是，我编辑文件不是在博客的那个文件夹里，而是在另外的一个知识库里，这就导致相对路径在一边可以，在另一边就不行；）
