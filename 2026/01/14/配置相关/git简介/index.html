git简介

Posted on 2026-01-14 In 配置相关

还是项目里需要用git管理一下。之前其实没怎么用过

常用命令：

git branch，查看本地的所有分支

git branch --show-current，展示我们当前所处的分支

git checkout branch-name，用于切换分支。执行这个命令，我们会切换到后面那个名字的分支

记录一下遇到的问题好了：

如果我们本地想创建一些分支，用来自己看/编辑/记录等，不想提交，怎么做？

就正常创建，编辑，完了可以add且commit，只要我们不push就没问题。并且这里的commit就类似于一个保存的操作（且如果我们从来不push，那么应该不会执行类似git push -u origin branch-name这样的操作，那也就不会和上游的远端仓库进行连接。即使真的不小心git push了，也不会传上去，会报错

如果要基于某个分支创建一个新的分支，进行改动，怎么做？

首先，切到我们要依赖的那个分支，然后执行git checkout -b new-branch-name即可。它就会基于当前的分支创建一个新的分支（但注意，这相当于分叉出一个派生分支。如果我们在新的那个分支里不add和commit，它就会和原来的分支一直保持一致，达不到我们想要的独立的效果）
