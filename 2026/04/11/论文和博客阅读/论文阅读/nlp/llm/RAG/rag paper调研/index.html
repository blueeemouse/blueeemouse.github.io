Self-RAG: Learning to Retrieve, Generate, and Critique through Self-Reflection（iclr 24 oral，1300+citations）

这篇论文针对的问题是，一般rag会不假思索地，在回答每个query之前都进行检索，同时对检索到的内容不加筛选（考虑到时间，早期的方法应该确实有这些问题吧）

论文就是想解决这一点。也很显然：并不是每次都一定要检索，如果模型内部的知识就足够回答了，那就没必要检索了；检索完之后也需要适当的进行筛选等操作。如果检索到太多无关的文档，就相当于对模型的干扰，可能降低效果

而论文大概的思路是，把模型的词表扩展一下，引入一些特殊的token。通过训练，模型学会何时生成这些特殊token，相应的掌握了对应token的能力（比如有的token代表检索，那么模型通过学会在合适的时候生成retrieve token，就学会了on-demand检索，而不是一上来就要检索）（这种用token控制模型行为的方法，似乎是很常见的。在23年那段时间好像就有其它领域的工作是这样做的。而最近特别火的agentic rag，本质上是不是也就是这样而已呢，只是可能引入了更多的特殊token而已？） # Adaptive-RAG: Learning to Adapt Retrieval-Augmented LLMs through Question Complexity（NAACL 24，300+citations） 它也是觉得不是所有query都需要检索，即使需要检索，所需的复杂度也不一样。因此它也考虑把query路由到不检索/单步检索/迭代式检索

它的方法就是训练一个轻量的分类器（经典T5-large），把来的query分成三个难度（对应上面的三种策略）（标签的收集也是很简单的，经典用所有候选方法来回答一遍，然后给标签。如果所有候选方法都答错了，那么我们就根据这个query所属的数据集的性质给标签）

值得注意的是，它的分类不是说提取特征然后送到分类器，而是让这个小的llm直接输出token/结果。这或许会不太一样

轻量是轻量，感觉做的工作也不多（而且它检索用的也是BM25啊，是为了极致追求轻量快速吗，毕竟BM25也不用训练） # Corrective Retrieval Augmented Generation（300+citations） 论文关注的是类似于rag的鲁棒性。当我们检索的文档里有错误的，不相关的（很常见），该怎么办？

论文是微调了一个评估器（T5-large，输入是query&文档，输出是一个介于1到-1之间的相关性分数。训练数据对是，输入为query+单个文档，标签是实现准备好的，为1/-1），用来评估每个检索文档和query的相关性。根据检索回来的这一批文档的置信度，有三个动作的触发机制：

  • Correct（至少一个文档置信度高）：对检索文档执行分解-重组精炼——将文档拆分为知识条，评估相关性，过滤无关条后重组关键信息。

  • Incorrect（所有文档置信度低）：丢弃检索结果，转用网络搜索作为补充知识源，通过查询重写获取权威网页内容。

  • Ambiguous（置信度居中）：结合内外部知识，同时利用精炼后的检索文档和网络搜索结果。

plug-and-play也是论文的一个卖点 # From Local to Global: A Graph RAG Approach to Query-Focused Summarization 真经典了，微软首次提出graphrag的paper

论文研究的问题

论文主要研究如何针对大规模文本语料库进行全局性查询聚焦摘要（Query-Focused Summarization, QFS），即如何回答需要理解整个数据集的“全局性理解（Sensemaking）”问题。

现有方法的不足

论文指出，目前的检索增强生成（RAG）和传统的 QFS 方法在处理大规模语料的全局问题时存在局限性：

传统向量 RAG（Vector RAG）： 擅长处理局部信息的检索，但无法支持需要对整个数据集进行全局理解的查询。

传统 QFS 方法： 无法扩展到典型 RAG 系统所处理的海量文本规模。

论文提出的方法如何解决问题

论文提出了 GraphRAG，通过构建图形索引和分层摘要来解决全局性问答任务：

构建图索引： 首先使用大语言模型（LLM）从源文档中提取实体知识图谱。

生成社区摘要： 对图中密切相关的实体进行社区检测，并预先生成各级社区的摘要。

Map-Reduce 处理： 在查询时，利用这些社区摘要通过 Map-Reduce 机制生成最终答案。

论文方法的创新性

GraphRAG 的创新性主要体现在将图索引与分层社区检测相结合，用于大规模文本的全局摘要：

利用图的模块化特性： 它是首个利用图的内在模块化（Modularity）和社区结构来对数据进行主题划分并生成分层摘要的方法。

分层索引结构： 通过递归地创建越来越全局的摘要，GraphRAG 能够跨越社区层级捕捉语料库的整体语义。

可扩展性： 相比于直接对源文本进行摘要，GraphRAG 在保持高全面性和多样性的同时，显著降低了处理全局查询所需的 Token 成本。 # MCTS-RAG: Enhancing Retrieval-Augmented Generation with Monte Carlo Tree Search（这个算比较新的？） 其实就是针对rag里检索和推理的独立性。一般检索过程是独立于推理的（检索到的内容也通常是直接塞给模型），这其实就是纯粹指望模型靠自己的能力来推理，没有做一些适配。所以论文想把推理和检索结合起来。它结合的方式就是说用MCTS搜索，只不过动作空间中加入了“retrieval”动作，还有其它的一些推理相关的动作（和self-rag有点类似，不过结合的方式不一样）
