实现Trie（前缀树）

Posted on 2025-05-08 In 算法面试题

原题链接在这

题目描述：请你实现 Trie 类：

- Trie() 初始化前缀树对象。
- void insert(String word) 向前缀树中插入字符串 word 。
- boolean search(String word) 如果字符串 word 在前缀树中，返回 true（即，在检索之前已经插入）；否则，返回 false 。
- boolean startsWith(String prefix) 如果之前已经插入的字符串 word 的前缀之一为 prefix ，返回 true ；否则，返回 false 。

示例：

输入
["Trie", "insert", "search", "search", "startsWith", "insert", "search"]
[[], ["apple"], ["apple"], ["app"], ["app"], ["app"], ["app"]]

输出
[null, null, true, false, true, null, true]

解释
Trie trie = new Trie();
trie.insert("apple");
trie.search("apple"); // 返回 True
trie.search("app"); // 返回 False
trie.startsWith("app"); // 返回 True
trie.insert("app");
trie.search("app"); // 返回 True

提示：
- 1 <= word.length, prefix.length <= 2000
- word 和 prefix 仅由小写英文字母组成
- insert、search 和 startsWith 调用次数 总计 不超过 3 * 104 次

思路：

这题明确说了是考Trie树，事实上也确实如此，没有别的技巧手法。所以这里就讲一下Trie树是什么，它的原理又是什么

灵茶山艾府的题解写的很好。他的引入也比较好，假设一个字符串只有两种字符，我们可以怎么表示这个字符串？不妨记其中一种字符为a，另一种字符为b，然后，比如字符串是abbaabb，从左到右，我们可以把它看成是一颗二叉树，a就是左儿子，b就是右儿子，这样，这个字符串可以被一个二叉树所唯一地表示。然后，这一题里主要有三个要求（初始化暂且先不管），第一个是insert，也就是向前缀树中插入字符串，那么在这里只有两种字符的例子中，就还是构造相应的二叉树即可；
第二个是search，也就是从根节点出发，搜索一下，有没有待搜索的字符串的对应的二叉树。考虑到有可能一些字符是另一些字符的前缀，例如，abab和ababaa，所以我们考虑给每个节点再附带上一个end属性，用来表征它是不是一个终止节点。从根节点出发，走到一个终止节点，就代表遍历了一个字符串。这样，search的时候，我们从根节点出发，一个一个搜索字符串里的字符，不仅要全部都出现，而且在搜索到了最后一个字符时，对应的节点必须end属性为true
第三个是startsWith，它要判断给定的一个字符串是否为已经出现过的某个字符串的前缀。与第二个要求对比，其实就是，搜索的时候，我们只要求所有的字符一定要出现，且不要求搜到最后一个字符的时候，end属性一定为true，此时我们不关注end属性了，为true或为false都行

那么，现在落实到题目里的要求，会出现所有的小写英文字母，那也就是26种字符，则我们构造的树就变成了26叉树了。至于insert，search以及startsWith三个操作，原理还是没变的

chap1.第三讲 双指针&位运算&离散化&区间合并

Posted on 2025-05-04 Edited on 2025-05-05 In algo

1.双指针

1.1.简介：这个其实更多是一种思想，或者说思路。它的主要作用是利用某种性质，进行优化，通常是把\(O(n^{2})\)的算法优化成\(O(n)\)。这里的性质，可以理解为某种单调性。比如，前面归并排序中，有一步要把两个有序的数组合并为一个有序的数组，其中就用到了双指针：两个指针从两个数组的起点开始，谁指向的元素更小，那个元素就是当前最小的元素，则放到合并后的大数组，并且相应指针往后移动一位。这里用到的单调性就是，两个数组都是有序的，当前指针后面的所有数都是大于等于当前指针指向的元素的，所以每次只需要比较一下当前两个指针指向的元素谁更小，不需要和后面的相比——这里就剩下了时间复杂度。所以最后两个指针都只需要遍历一遍数组，故时间复杂度是\(O(n)\)
